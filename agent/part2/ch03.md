# 第三章 Claude 生态中的 Agent 全景

## 五种 Agent 形态

Claude 生态中有五种不同的 Agent 形态，从简单到复杂：

```
Level 1: Claude Code 单 Agent       ← 你现在最常用的
Level 2: Subagents（子代理）         ← 并行工作者
Level 3: Agent Teams（代理团队）     ← 互相协作的团队
Level 4: Managed Agents（远程代理）  ← 云端自动执行
Level 5: Agent SDK（编程式）         ← 用代码构建自定义 Agent
```

## Level 1: Claude Code 单 Agent

这就是你平时用的 `claude` 命令：

```bash
claude
> 帮我修复这个 bug
```

一个 Agent、一个对话、一个任务。

**工作流程**：
```
你 → Claude → [读代码] → [分析] → [修改] → [测试] → 结果
```

**适合**：
- 日常编码问答
- 单文件修改
- 简单的多文件操作（<5 个文件）

**限制**：
- 串行执行（一次只能做一件事）
- 上下文有限（大项目读不完）
- 单一视角（没有"第二意见"）

## Level 2: Subagents

主 Agent 可以 spawn 子 Agent 来并行处理任务：

```
你 → 主 Agent
        ├──→ Subagent A：搜索安全问题
        ├──→ Subagent B：分析性能
        └──→ Subagent C：检查测试覆盖
        ← 三个结果汇总
```

### 使用方式

```
用三个 subagent 并行分析这个项目：
1. 一个检查安全问题
2. 一个分析性能瓶颈
3. 一个评估测试覆盖率

各自完成后汇报给我。
```

### 关键特性

- 每个 Subagent 有**独立的上下文**（互不干扰）
- Subagent 完成后结果返回给主 Agent
- Subagent 之间**不能互相通信**
- **默认可用**，不需要额外配置

### 最佳场景

| 场景 | 为什么用 Subagent |
|------|------------------|
| 并行搜索多个目录 | 各自独立，不需要通信 |
| 多维度代码分析 | 结果汇总即可 |
| 隔离探索型任务 | 防止大量文件内容污染主 Agent 上下文 |
| 用便宜模型做简单任务 | `CLAUDE_CODE_SUBAGENT_MODEL=haiku` |

## Level 3: Agent Teams

多个 Agent 组成团队，有 Leader 协调，成员间可以直接通信：

```
你 → Team Lead（协调者）
        ├──→ Teammate A（前端）
        │         ↕ 通信
        ├──→ Teammate B（后端）
        │         ↕ 通信
        └──→ Teammate C（测试）
```

### 使用方式

```
# 先启用
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1

# 在 Claude Code 中
创建一个 3 人团队来开发用户设置功能：
1. frontend-dev：负责 React 组件
2. backend-dev：负责 API endpoints
3. test-dev：负责测试

frontend-dev 和 backend-dev 需要协商接口定义。
```

### 与 Subagent 的核心区别

| 维度 | Subagent | Agent Teams |
|------|----------|-------------|
| 通信 | ❌ 只向主 Agent 汇报 | ✅ 成员间直接通信 |
| 协调 | 主 Agent 管理 | 共享任务列表 + Mailbox |
| 关系 | 上下级 | 同事 |
| 适合 | 独立任务 | 需要协作的任务 |

### 何时从 Subagent 升级到 Agent Teams

问自己：**并行的任务之间需要互相"说话"吗？**

- 不需要 → Subagent
- 需要 → Agent Teams

## Level 4: Managed Agents（远程代理）

Agent 在云端执行，你不需要保持本地运行：

```bash
# 创建定时任务
/schedule create --cron "0 9 * * 1-5" --prompt "生成每日代码质量报告"
```

### 使用方式

```
/schedule create --cron "0 9 * * 1" --prompt "
检查本周的依赖安全漏洞，
如果有 Critical 级别的，在 Slack #security 频道通知
"
```

### 适合场景

- 定期报告生成
- 自动化安全扫描
- 定时数据分析
- 无人值守的维护任务

## Level 5: Agent SDK（编程式）

用代码构建你自己的 Agent：

```typescript
import { createAgent } from '@anthropic-ai/claude-code';

const reviewer = createAgent({
  model: 'claude-sonnet-4-6',
  systemPrompt: '你是代码审查专家...',
  tools: ['Read', 'Grep'],
});

const result = await reviewer.run('审查 src/auth/ 的安全性');
```

### 适合场景

- 嵌入到你的产品中
- CI/CD 自动化流水线
- 自定义的 Slack/Discord Bot
- 需要精确控制 Agent 行为的场景

## 选择决策树

```
你的任务是什么？
│
├─ 日常开发，一个人干 → Level 1（单 Agent）
│
├─ 需要并行但各自独立 → Level 2（Subagent）
│
├─ 需要多人协作讨论 → Level 3（Agent Teams）
│
├─ 需要定时自动执行 → Level 4（Managed Agents）
│
└─ 需要嵌入产品/CI → Level 5（Agent SDK）
```

## 组合使用

这五个级别不是互斥的，可以组合：

```
Level 3 Agent Teams
├── Teammate A
│   └── Level 2 Subagent（做局部探索）
├── Teammate B
│   └── Level 2 Subagent（做性能测试）
└── Level 4 Managed Agent（每天自动汇总进度）
```

最强大的团队 = Agent Teams（协作）+ Subagent（并行）+ MCP（外部连接）。
