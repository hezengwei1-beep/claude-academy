# 第一章 什么是 Agent

## 一句话定义

**Agent = LLM + 工具 + 自主决策循环**

普通的 LLM（大语言模型）只能"想"——你给它文本，它返回文本。

Agent 在此基础上增加了两个关键能力：
1. **工具**：Agent 可以调用外部工具（读文件、查数据库、发 API 请求）
2. **循环**：Agent 可以根据工具返回的结果决定下一步做什么，形成"思考 → 行动 → 观察 → 再思考"的循环

## Agent vs ChatGPT vs Claude Code

```
ChatGPT（传统聊天）:
  用户提问 → LLM 回答 → 结束

Claude Code（Agent）:
  用户描述任务
    → LLM 思考"我需要先读文件了解代码结构"
    → 调用 Read 工具读取文件
    → LLM 观察文件内容，思考"这里有个 bug"
    → 调用 Edit 工具修改代码
    → LLM 思考"改完了，需要验证"
    → 调用 Bash 工具运行测试
    → LLM 观察"测试通过"
    → 向用户报告"bug 已修复"
```

**区别不是智能程度，而是工作模式**：
- ChatGPT = 一问一答（单轮）
- Agent = 自主规划和执行（多轮循环）

## Agent 的核心组件

```
┌─────────────────────────────────────────┐
│                Agent                     │
│                                          │
│  ┌──────────────┐  ┌─────────────────┐  │
│  │   LLM 大脑    │  │  System Prompt  │  │
│  │  (思考和决策)  │  │  (角色和规则)   │  │
│  └──────┬───────┘  └─────────────────┘  │
│         │                                │
│  ┌──────▼──────────────────────────┐    │
│  │        决策循环                   │    │
│  │  思考 → 选择工具 → 执行 → 观察   │    │
│  │         ↑                  │      │    │
│  │         └──────────────────┘      │    │
│  └──────────────────────────────────┘    │
│                                          │
│  ┌──────────────────────────────────┐    │
│  │           工具箱                   │    │
│  │  Read │ Write │ Bash │ Grep │ ...│    │
│  └──────────────────────────────────┘    │
│                                          │
│  ┌──────────────────────────────────┐    │
│  │           记忆系统                 │    │
│  │  对话历史 │ 文件 │ Memory          │    │
│  └──────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

### 1. LLM 大脑

Agent 的"思考引擎"。它负责：
- 理解用户的任务
- 决定下一步做什么
- 选择用什么工具
- 理解工具返回的结果
- 判断任务是否完成

在 Claude 生态中，LLM 就是 Claude Opus/Sonnet/Haiku。

### 2. System Prompt（角色定义）

告诉 Agent "你是谁、你会什么、你的规则是什么"：

```
你是一个代码审查专家。
你可以使用 Read 和 Grep 工具来查看代码。
你不能修改任何文件。
发现问题时按严重度排序报告。
```

### 3. 工具箱

Agent 可以调用的外部能力：

| 工具类型 | 示例 | 作用 |
|----------|------|------|
| 文件操作 | Read, Write, Edit | 读写代码和文件 |
| 命令执行 | Bash | 运行测试、安装依赖 |
| 代码搜索 | Grep, Glob | 在代码库中查找 |
| 外部服务 | MCP Servers | 查数据库、调 API |
| 子代理 | Agent/SubAgent | 委派子任务 |

### 4. 决策循环（Agent Loop）

这是 Agent 的核心机制：

```
while 任务未完成:
    1. 思考：当前状态是什么？下一步该做什么？
    2. 决策：选择一个工具（或直接回答用户）
    3. 执行：调用工具，获取结果
    4. 观察：分析结果，更新理解
    5. 判断：任务完成了吗？
        - 完成 → 向用户报告
        - 未完成 → 回到步骤 1
```

### 5. 记忆系统

Agent 需要记住：
- **短期记忆**：当前对话的上下文（自动管理）
- **中期记忆**：会话之间的恢复（/resume）
- **长期记忆**：跨会话的知识（Memory 系统、文件）

## Agent 的能力边界

Agent 不是万能的。理解边界和理解能力一样重要：

### Agent 擅长

- 有明确目标的多步骤任务
- 可以用工具验证结果的任务（如：改代码 → 跑测试）
- 有清晰成功标准的任务

### Agent 不擅长

- 需要创造力和主观判断的任务（"设计一个好看的 Logo"）
- 没有明确完成标准的开放式任务（"让这段代码更好"）
- 需要物理世界交互的任务（除了 Computer Use）
- 需要最新信息但没有联网工具的任务

### Agent 的风险

| 风险 | 说明 | 缓解 |
|------|------|------|
| 循环 | Agent 陷入"修 A 破 B、修 B 破 A"的死循环 | 设置最大循环次数 |
| 漂移 | Agent 偏离原始目标 | 明确的 system prompt |
| 幻觉 | Agent 声称完成了实际没完成的任务 | 用工具验证（跑测试） |
| 失控 | Agent 做了不该做的操作 | 权限控制 + 人工确认 |

## Agent 的演化路径

```
Level 0: Chatbot
  → 一问一答，无工具

Level 1: Tool-Augmented LLM
  → 能调用工具，但只执行一步
  → 示例：ChatGPT 的 Function Calling

Level 2: Single Agent
  → 自主循环，多步骤完成任务
  → 示例：Claude Code（单 agent 模式）

Level 3: Multi-Agent
  → 多个 Agent 协作
  → 示例：Claude Code Agent Teams

Level 4: Agent Ecosystem
  → Agent 可以发现和调用其他 Agent
  → MCP + Agent Teams + Managed Agents
  → 2026 年正在构建中
```

你现在所处的时代：**Level 2-3 的过渡期**。单 Agent 已经成熟，多 Agent 正在快速发展。
