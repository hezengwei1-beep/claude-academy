# 第十章 环境配置与首次启动

## 版本要求

Agent Teams 需要 **Claude Code v2.1.32 或更高版本**。

检查当前版本：
```bash
claude --version
```

更新到最新版：
```bash
npm update -g @anthropic-ai/claude-code
```

## 启用 Agent Teams

Agent Teams 是实验性功能，默认关闭。启用方式有两种：

### 方式一：环境变量（临时）

```bash
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
claude
```

这种方式只对当前 shell 会话有效。

### 方式二：settings.json（永久）

编辑 `~/.claude/settings.json`：

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

这种方式永久生效，推荐使用。

## 显示模式配置

Agent Teams 支持两种显示模式，决定了 teammates 如何在你的终端中展示。

### In-process 模式（默认）

所有 teammates 在主终端窗口内运行。你通过快捷键在它们之间切换。

```json
{
  "teammateMode": "in-process"
}
```

**优点**：
- 无需额外工具
- 在任何终端中都能使用
- 设置零成本

**缺点**：
- 同一时间只能看到一个 teammate 的输出
- 无法同时观��多个 teammate 的实时进度

**快捷键**：
| 快捷键 | 作用 |
|--------|------|
| `Shift+Down` | 切换到下���个 teammate |
| `Shift+Up` | 切换���上一个 teammate |
| `Ctrl+T` | 查看/切换任���列表 |
| `Enter` | 进入当前 teammate 的会话详情 |
| `Escape` | 中断当前 teammate 的操作 |

### Split-pane 模式（tmux/iTerm2）

每个 teammate 在独立的终端窗格中显示，可以同时看到所有 teammate 的输出。

**tmux 配置**：

```json
{
  "teammateMode": "tmux"
}
```

需要先安装 tmux：
```bash
brew install tmux
```

**iTerm2 配置**：

```json
{
  "teammateMode": "auto"
}
```

`auto` 模式会自动检测你使用的终端。如果是 iTerm2，会使用原生的 split pane。

**优点**：
- 同时看到所有 teammate 的实时输出
- 更直观的监控体验

**缺点**：
- 需要安装额外工具
- VS Code 终端、Windows Terminal、Ghostty 不支持

**注意**：如果你用 VS Code 内置终端，只能用 in-process 模式。

## 首次启动验证

配置完成后，验证 Agent Teams 是否正常工作：

### 步骤 1：启动 Claude Code

```bash
claude
```

### 步骤 2：创建一个最小团队

在 Claude Code 中输入：

```
创建一个 2 人研究团队：
- researcher-a：查看 README.md 文件并总结项目概况
- researcher-b：查看项目的目录结构并报告

两人完成后各自在任务列表中标记完成。
```

### 步骤 3：观察

你应该看到：
1. Claude 创建了两个 teammates
2. 两个 teammates 各自开始��作
3. 任务列表中出现对应的任务
4. 完成后你可以看到汇总结果

如果以上步骤顺利，你的 Agent Teams 环境已经就绪。

## 完整的 settings.json 示例

以下是一个推荐的 Agent Teams 配置：

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  },
  "teammateMode": "in-process"
}
```

如果你使用 tmux 并且想要更好的可视化：

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  },
  "teammateMode": "tmux"
}
```

## 常见配置问题排查

### 问题 1：启动后没有 Agent Teams 相关功能

**排查**：
```bash
echo $CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS
```
如果输出为空，环境变量没有生效。检查 settings.json 的格���是否正确。

### 问题 2：tmux 模式下窗格不显示

**排查**：
- 确认 tmux 已安装：`which tmux`
- 确认在 tmux 会话外启动 Claude Code（不要在已有的 tmux 会话中嵌套）
- 或者先创建 tmux 会话再启动：`tmux new -s claude && claude`

### 问题 3：teammate 创建失败

**排查**：
- 确认 Claude Code 版本 >= 2.1.32
- 检查网络连接（每个 teammate 需要独立的 API ���接）
- 检查 API 配额是否充足（每个 teammate 消耗独立的 token）

### 问题 4：任务列表不更新

**已知限制**：任务列表的同步有短暂延迟，通常在几秒内更新。如果长时间不更新，尝试按 `Ctrl+T` 手动刷新。
