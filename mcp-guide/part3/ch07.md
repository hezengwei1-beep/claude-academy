
# 第七章 用 TypeScript 从零构建 MCP Server

## 前置条件

- Node.js >= 18
- TypeScript 基础知识
- 一个你想让 Claude 连接的服务（本章用天气 API 做示例）

## 项目初始化

```bash
mkdir weather-mcp-server && cd weather-mcp-server
npm init -y
npm install @modelcontextprotocol/sdk zod
npm install -D typescript @types/node
npx tsc --init
```

编辑 `tsconfig.json`：
```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "Node16",
    "moduleResolution": "Node16",
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true
  },
  "include": ["src/**/*"]
}
```

编辑 `package.json`，添加：
```json
{
  "type": "module",
  "bin": {
    "weather-mcp": "./dist/index.js"
  },
  "scripts": {
    "build": "tsc",
    "start": "node dist/index.js"
  }
}
```

## 编写 Server

创建 `src/index.ts`：

```typescript
#!/usr/bin/env node

import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";

// 创建 Server 实例
const server = new McpServer({
  name: "weather-server",
  version: "1.0.0",
});

// 定义工具：获取天气
server.tool(
  "get_weather",
  "获取指定城市的当前天气信息",
  {
    city: z.string().describe("城市名称，如 Beijing, Tokyo, New York"),
    unit: z.enum(["celsius", "fahrenheit"]).default("celsius").describe("温度单位"),
  },
  async ({ city, unit }) => {
    // 实际项目中这里调用真实的天气 API
    // 这里用模拟数据演示
    const mockWeather = {
      city,
      temperature: unit === "celsius" ? 22 : 72,
      unit,
      condition: "晴",
      humidity: 45,
      wind: "东北风 3 级",
    };

    return {
      content: [
        {
          type: "text" as const,
          text: JSON.stringify(mockWeather, null, 2),
        },
      ],
    };
  }
);

// 定义工具：获取天气预报
server.tool(
  "get_forecast",
  "获取指定城市未来几天的天气预报",
  {
    city: z.string().describe("城市名称"),
    days: z.number().min(1).max(7).default(3).describe("预报天数，1-7"),
  },
  async ({ city, days }) => {
    const forecast = Array.from({ length: days }, (_, i) => ({
      day: `第 ${i + 1} 天`,
      high: 20 + Math.floor(Math.random() * 10),
      low: 10 + Math.floor(Math.random() * 10),
      condition: ["晴", "多云", "小雨", "阴"][Math.floor(Math.random() * 4)],
    }));

    return {
      content: [
        {
          type: "text" as const,
          text: `${city} 未来 ${days} 天预报：\n${JSON.stringify(forecast, null, 2)}`,
        },
      ],
    };
  }
);

// 定义资源：支持的城市列表
server.resource(
  "cities://supported",
  "cities://supported",
  async (uri) => ({
    contents: [
      {
        uri: uri.href,
        mimeType: "application/json",
        text: JSON.stringify(["Beijing", "Shanghai", "Tokyo", "New York", "London"]),
      },
    ],
  })
);

// 定义 Prompt 模板
server.prompt(
  "weather_report",
  "生成一份完整的天气报告",
  [
    { name: "city", description: "城市名称", required: true },
  ],
  async ({ city }) => ({
    messages: [
      {
        role: "user" as const,
        content: {
          type: "text" as const,
          text: `请为 ${city} 生成一份详细的天气报告，包括当前天气和未来 3 天预报。使用 get_weather 和 get_forecast 工具获取数据，然后整理成可读的报告格式。`,
        },
      },
    ],
  })
);

// 启动 Server
async function main() {
  const transport = new StdioServerTransport();
  await server.connect(transport);
  console.error("Weather MCP Server started"); // 注意：用 stderr，不能用 stdout
}

main().catch(console.error);
```

## 构建和测试

### 构建

```bash
npm run build
```

### 用 Inspector 测试

```bash
npx @modelcontextprotocol/inspector node dist/index.js
```

Inspector 会打开一个网页界面，你可以：
1. 在 **Tools** 标签页看到 `get_weather` 和 `get_forecast`
2. 点击工具测试调用
3. 在 **Resources** 标签页看到支持的城市列表
4. 在 **Prompts** 标签页看到天气报告模板

### 连接到 Claude

在 `claude_desktop_config.json` 或 `.mcp.json` 中添加：

```json
{
  "mcpServers": {
    "weather": {
      "command": "node",
      "args": ["/path/to/weather-mcp-server/dist/index.js"]
    }
  }
}
```

重启 Claude Desktop 或 Claude Code，然后试试：

```
北京今天天气怎么样？
```

## 关键注意事项

### 1. stdout 只给协议用

Server 的 stdout 只能输出 MCP 协议消息。日志、调试信息必须写到 **stderr**：

```typescript
// 正确
console.error("Debug info");

// 错误——会破坏协议通信
console.log("Debug info");
```

### 2. 输入验证

用 Zod 做参数校验。永远不要信任 AI 传来的参数：

```typescript
server.tool(
  "query_database",
  "执行数据库查询",
  {
    sql: z.string()
      .refine(s => s.trim().toUpperCase().startsWith("SELECT"), {
        message: "只允许 SELECT 查询"
      }),
  },
  async ({ sql }) => { /* ... */ }
);
```

### 3. 错误处理

工具调用失败时返回 `isError: true`：

```typescript
try {
  const result = await fetchWeather(city);
  return { content: [{ type: "text", text: result }] };
} catch (error) {
  return {
    content: [{ type: "text", text: `获取天气失败: ${error.message}` }],
    isError: true,
  };
}
```

### 4. 工具描述要写好

AI 根据工具的 name 和 description 决定何时调用它。描述要准确、具体：

```typescript
// 差
"get_data"  // AI 不知道这是什么 data

// 好
"get_current_weather"  // 清楚是获取当前天气
```
