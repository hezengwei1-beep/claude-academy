# 第九章 调试、测试与发布

## MCP Inspector：核心调试工具

Inspector 是 MCP 官方提供的可视化调试工具。

### 安装和使用

```bash
# 调试 TypeScript Server
npx @modelcontextprotocol/inspector node dist/index.js

# 调试 Python Server
npx @modelcontextprotocol/inspector uv run server.py

# 调试已安装的 npm Server
npx @modelcontextprotocol/inspector npx @modelcontextprotocol/server-filesystem /tmp
```

### Inspector 界面

Inspector 打开后提供四个标签页：

1. **Tools**：列出所有工具，可以手动输入参数测试调用
2. **Resources**：列出所有资源，可以查看内容
3. **Prompts**：列出所有提示词模板，可以测试生成
4. **Notifications**：实时显示 Server 的通知和日志

### 调试流程

```
1. 用 Inspector 启动你的 Server
2. 在 Tools 标签页测试每个工具
   - 输入正常参数 → 验证输出正确
   - 输入边界值 → 验证错误处理
   - 输入空值/异常值 → 验证不会崩溃
3. 在 Resources 标签页验证资源可读取
4. 在 Prompts 标签页验证模板生成正确
5. 确认没问题后再连接到 Claude
```

## 日志调试

### stdio Server 日志

stdio Server 的 stdout 被协议占用，日志必须写到 stderr：

```typescript
// TypeScript
console.error("[DEBUG] Processing request:", params);
```

```python
# Python
import sys
print("[DEBUG] Processing request:", params, file=sys.stderr)
```

### Claude Desktop 日志

```bash
# macOS - 查看 MCP 相关日志
ls ~/Library/Logs/Claude/mcp*.log
tail -f ~/Library/Logs/Claude/mcp-server-*.log
```

### 结构化日志

使用 MCP 的通知机制发送日志：

```typescript
server.sendLoggingMessage({
  level: "info",
  data: "Processing weather request for Beijing",
});
```

## 单元测试

### TypeScript 测试

```typescript
// __tests__/server.test.ts
import { describe, it, expect } from "vitest";

describe("Weather Server", () => {
  it("should return weather for valid city", async () => {
    const result = await getWeather("Beijing", "celsius");
    expect(result).toHaveProperty("temperature");
    expect(result).toHaveProperty("condition");
  });

  it("should handle invalid city gracefully", async () => {
    const result = await getWeather("", "celsius");
    expect(result.isError).toBe(true);
  });
});
```

### Python 测试

```python
# test_server.py
import pytest
from server import load_todos, save_todos

def test_add_todo():
    # 清空
    save_todos([])
    
    from server import add_todo
    result = add_todo("测试任务", "high")
    assert "已添加" in result
    
    todos = load_todos()
    assert len(todos) == 1
    assert todos[0]["title"] == "测试任务"
    assert todos[0]["priority"] == "high"

def test_complete_todo():
    save_todos([{"id": 1, "title": "测试", "priority": "medium", "done": False}])
    
    from server import complete_todo
    result = complete_todo(1)
    assert "已完成" in result
    
    todos = load_todos()
    assert todos[0]["done"] is True
```

## 集成测试

用 MCP Client SDK 做端到端测试：

```typescript
import { Client } from "@modelcontextprotocol/sdk/client/index.js";
import { StdioClientTransport } from "@modelcontextprotocol/sdk/client/stdio.js";

const transport = new StdioClientTransport({
  command: "node",
  args: ["dist/index.js"],
});

const client = new Client({ name: "test-client", version: "1.0.0" });
await client.connect(transport);

// 测试工具列表
const tools = await client.listTools();
expect(tools.tools.length).toBeGreaterThan(0);

// 测试工具调用
const result = await client.callTool("get_weather", { city: "Beijing" });
expect(result.content[0].text).toContain("Beijing");

await client.close();
```

## 发布

### 发布到 npm（TypeScript）

```bash
# 确保 package.json 正确
npm publish
```

用户安装：
```json
{ "command": "npx", "args": ["-y", "your-package-name"] }
```

### 发布到 PyPI（Python）

```bash
# 使用 uv
uv build
uv publish
```

用户安装：
```json
{ "command": "uvx", "args": ["your-package-name"] }
```

### 提交到 MCP Registry

MCP Registry 是官方的 Server 发现平台（预览阶段）：

1. 在 `registry.modelcontextprotocol.io` 注册
2. 使用反向 DNS 命名（如 `io.github.username/server-name`）
3. 通过 DNS 验证确认命名空间所有权
4. 提交 Server 元数据

### 提交到 awesome-mcp-servers

在 `github.com/punkpeye/awesome-mcp-servers` 提交 PR，将你的 Server 添加到对应分类。
