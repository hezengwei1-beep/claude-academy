
# 第六章 实战：用 MCP 打造个人工作流

## 场景一：研究助手工作流

**目标**：让 Claude 能搜索网页、读取本地文件、保存笔记。

### 配置

```json
{
  "mcpServers": {
    "fetch": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-fetch"]
    },
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "~/Research"]
    },
    "memory": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-memory"]
    }
  }
}
```

### 使用方式

```
帮我研究 "MCP 协议的最新进展"：
1. 搜索相关网页获取最新信息
2. 整理成结构化笔记保存到 ~/Research/mcp-notes.md
3. 记住关键发现，下次我问的时候能想起来
```

Claude 会依次调用 fetch（获取网页）→ filesystem（保存文件）→ memory（记忆关键信息）。

## 场景二：开发者日常工作流

**目标**：Claude 能操作 GitHub、查数据库、发 Slack 消息。

### 配置

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_..." }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@mcp/server-postgres"],
      "env": { "DATABASE_URL": "postgresql://readonly:pass@localhost/myapp" }
    },
    "slack": {
      "command": "npx",
      "args": ["-y", "@mcp/server-slack"],
      "env": { "SLACK_BOT_TOKEN": "xoxb-..." }
    }
  }
}
```

### 使用场景

```
查看 my-org/my-repo 上最近的 5 个 open Issues，
然后查数据库看每个 Issue 对应的用户有多少，
最后在 #dev-updates Slack 频道发一份摘要。
```

Claude 依次调用三个 MCP Server，完成跨系统的工作流。

## MCP 工作流设计原则

### 1. 最小权限

每个 Server 只配置必需的权限。数据库用只读用户，GitHub 用最小 scope 的 token。

### 2. 组合而非堆叠

不要安装你不需要的 Server。3-5 个精选的 Server 组合通常覆盖 80% 的需求。

### 3. 安全第一

- 敏感操作（写入、删除、发送消息）设为需要用户确认
- API Key 存入环境变量，不要硬编码
- 定期轮换 token

### 4. 渐进式构建

从一个 Server 开始，验证好用后再加新的。不要一开始就搭一个 10 个 Server 的"超级配置"。

## 调试技巧

### 查看已连接的 Servers

在 Claude Code 中：
```
/mcp
```

在 Claude Desktop 中：点击输入框旁的 MCP 图标。

### 查看 Server 日志

```bash
# macOS Claude Desktop 日志
tail -f ~/Library/Logs/Claude/mcp*.log
```

### 使用 Inspector 调试

```bash
npx @modelcontextprotocol/inspector npx @modelcontextprotocol/server-filesystem ~/Documents
```

Inspector 提供可视化界面查看 Server 的所有工具、资源和 prompts。
