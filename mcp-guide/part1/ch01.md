
# 第一章 MCP 是什么：让 AI 连接万物的协议

## 一个类比

你有没有想过，为什么你的 USB 线可以接鼠标、键盘、硬盘、打印机——不管什么品牌、什么设备？因为它们都遵循 USB 协议。

**MCP（Model Context Protocol）就是 AI 世界的 USB 协议。**

它定义了一个标准接口，让 AI 模型（如 Claude）可以连接任何外部工具和数据源——数据库、API、文件系统、SaaS 服务、内部系统——而不需要为每个服务单独写集成代码。

## 正式定义

MCP 是 Anthropic 于 2024 年底发布的一个**开放协议**，全称 Model Context Protocol。它定义了 AI 模型与外部数据源、工具之间的标准化通信方式。

核心理念：**与其让每个 AI 应用都自己写集成代码，不如定义一个通用协议，让工具和数据源只需要实现一次，就能被所有支持 MCP 的 AI 使用。**

## MCP 的三层架构

```
┌─────────────────────────────────────────────┐
│  MCP Host（宿主）                            │
│  Claude Desktop / Claude Code / 你的应用     │
│                                             │
│  ┌─────────────────────────────────────┐    │
│  │  MCP Client（客户端）                │    │
│  │  内置在 Host 中，处理协议通信         │    │
│  └────────┬────────────────────────────┘    │
└───────────┼─────────────────────────────────┘
            │  MCP 协议（JSON-RPC 2.0）
            │
┌───────────▼─────────────────────────────────┐
│  MCP Server（服务器）                        │
│  每个服务器提供特定的工具/数据               │
│                                             │
│  例如：                                     │
│  • GitHub Server → 读写 Issues/PRs          │
│  • PostgreSQL Server → 查询数据库            │
│  • Filesystem Server → 读写本地文件          │
│  • Slack Server → 发送/读取消息              │
└─────────────────────────────────────────────┘
```

### MCP Host

"宿主"——运行 AI 模型的应用。Claude Desktop、Claude Code、或者你自己用 SDK 构建的应用都是 Host。

### MCP Client

内置在 Host 中的"翻译官"，负责把 AI 的请求翻译成 MCP 协议消息发给 Server，再把 Server 的响应翻译回来给 AI。

### MCP Server

"服务提供者"——每个 Server 封装一个特定的能力。GitHub Server 让 AI 能操作 GitHub，PostgreSQL Server 让 AI 能查数据库。

## MCP Server 提供的三种能力

每个 MCP Server 可以暴露三种类型的能力：

| 类型 | 说明 | 示例 |
|------|------|------|
| **Tools（工具）** | AI 可以调用的函数 | `create_issue`、`query_database`、`send_message` |
| **Resources（资源）** | AI 可以读取的数据 | 文件内容、数据库记录、API 响应 |
| **Prompts（提示词模板）** | 预定义的交互模板 | "基于这个 schema 生成查询"、"总结这个 PR" |

## 为什么需要 MCP

### 没有 MCP 的世界

```
Claude Desktop  ←→  自定义 GitHub 集成代码
Claude Desktop  ←→  自定义 Slack 集成代码
Claude Desktop  ←→  自定义数据库集成代码
Claude Code     ←→  另一套 GitHub 集成代码
Claude Code     ←→  另一套 Slack 集成代码
你的应用        ←→  又一套 GitHub 集成代码
...

N 个 Host × M 个服务 = N×M 个集成
```

### 有了 MCP 的世界

```
Claude Desktop  ──┐
Claude Code     ──┤── MCP 协议 ──→  GitHub MCP Server
你的应用        ──┘                 Slack MCP Server
                                    PostgreSQL MCP Server
                                    ...

N 个 Host + M 个 Server = N+M 个实现
```

复杂度从乘法降到了加法。这就是标准协议的力量。

### 与传统 API 集成的对比

| 维度 | 传统 API 集成 | MCP |
|------|-------------|-----|
| 每个服务需要 | 自定义代码 | 一个标准 MCP Server |
| AI 理解能力 | 需要教 AI 每个 API 的用法 | Server 自描述，AI 自动理解 |
| 安全控制 | 各自实现 | 协议级统一权限模型 |
| 发现机制 | 无（硬编码） | Server 自动暴露能力列表 |
| 复用性 | 低（与特定 Host 绑定） | 高（任何 MCP Host 都能用） |

## MCP 的生态现状

截至 2026 年，MCP 生态已经非常活跃：

- **官方 MCP Servers**：Anthropic 官方维护约 20+ 个 Server
- **社区 MCP Servers**：数百个社区开发的 Server
- **支持的 Host**：Claude Desktop、Claude Code、Cursor、Windsurf、Cline 等
- **SDK**：TypeScript SDK、Python SDK、Kotlin SDK、C# SDK 等
- **协议版本**：基于 JSON-RPC 2.0，持续演进中
