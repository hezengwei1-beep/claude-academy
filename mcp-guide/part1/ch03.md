# 第三章 5 分钟安装第一个 MCP Server

## 目标

本章结束时，你将有一个可用的 MCP Server 连接到 Claude，让 Claude 能直接操作你的文件系统。

## 在 Claude Desktop 中配置

### Step 1：找到配置文件

macOS 配置文件位置：
```
~/Library/Application Support/Claude/claude_desktop_config.json
```

如果文件不存在，创建一个。

### Step 2：添加 Filesystem Server

编辑 `claude_desktop_config.json`：

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/你的用户名/Documents"
      ]
    }
  }
}
```

这告诉 Claude Desktop：
- 启动一个名为 `filesystem` 的 MCP Server
- 用 `npx` 运行 `@modelcontextprotocol/server-filesystem` 包
- 允许它访问 `~/Documents` 目录

### Step 3：重启 Claude Desktop

完全退出（Cmd+Q）再重新打开。

### Step 4：验证

在 Claude Desktop 的对话中输入：

```
列出 ~/Documents 目录下的所有文件
```

如果 Claude 能列出你的文件，MCP Server 安装成功。

## 在 Claude Code 中配置

### 方式一：项目级配置

在项目根目录创建 `.mcp.json`：

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "."
      ]
    }
  }
}
```

### 方式二：全局配置

编辑 `~/.claude/settings.json`：

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/你的用户名/Documents"
      ]
    }
  }
}
```

### 方式三：交互式添加

在 Claude Code 中使用 `/mcp` 命令：

```
/mcp add filesystem npx -y @modelcontextprotocol/server-filesystem ~/Documents
```

### 验证

```
/mcp
```

查看已连接的 MCP Servers 列表。

## 常见问题

### npx 找不到

确保 Node.js 已安装：
```bash
node --version  # 应 >= 18
npx --version
```

### Server 连接失败

检查配置文件的 JSON 格式是否正确（常见错误：多了逗号、漏了引号）。

### 权限问题

Filesystem Server 只能访问你在 args 中指定的目录。如果需要访问其他目录，修改路径参数。
