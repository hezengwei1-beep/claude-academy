
# 第七章 场景实战：写作、翻译与数据分析

## 写作场景

### 技术文档

```xml
<task>
为以下 API 编写技术文档
</task>

<api_spec>
POST /api/users
请求体: {"name": string, "email": string, "role": "admin"|"user"}
响应 201: {"id": number, "name": string, "email": string, "role": string, "created_at": string}
响应 400: {"error": string}
响应 409: {"error": "Email already exists"}
</api_spec>

<doc_format>
包含以下部分：
1. 概述（一句话说明用途）
2. 请求参数表格
3. 响应格式（成功和失败）
4. curl 示例
5. TypeScript 示例
6. 常见错误和排查
</doc_format>

<style>
- 简洁直接
- 代码示例可直接复制运行
- 使用表格而非长段文字
</style>
```

### 商业邮件

```
请帮我写一封给客户的项目进度更新邮件。

<context>
- 客户：Acme Corp 的 CTO 张总
- 项目：CRM 系统重构
- 关系：合作两年，关系不错
- 进度：完成 70%，比计划延迟一周
- 延迟原因：第三方 API 接口变更，需要适配
- 好消息：性能比预期好 30%
</context>

<tone>
专业但不生硬，坦诚但不焦虑。
先说好消息，再说延迟，给出解决方案。
</tone>

<length>200-300 字</length>
```

### 创意写作

```
写一篇关于 "AI 结对编程" 的技术博客。

<audience>有 3-5 年经验的后端开发者</audience>

<angle>
不要写"AI 多牛"的吹捧文。
写真实体验：什么有效、什么无效、什么让人意外。
</angle>

<structure>
- 标题（吸引人但不标题党）
- 引入（一个具体的故事或场景，30 字以内）
- 正文 3-4 个要点
- 结论（一句话总结）
</structure>

<length>1500-2000 字</length>
```

## 翻译场景

### 技术文档翻译

```xml
<task>翻译以下技术文档</task>

<source language="en">
[英文内容]
</source>

<rules>
1. 专业术语保留英文，括号内附中文首次出现时的翻译
   例：Model Context Protocol (模型上下文协议)
2. 后续出现直接用英文缩写（MCP）
3. 代码块不翻译
4. 保持 Markdown 格式
5. 语气正式但不生硬
6. 不要意译到失真，也不要逐字死译
</rules>

<target language="zh-CN"/>
```

### 学术论文辅助翻译

```xml
<task>翻译以下学术论文摘要</task>

<source>
[论文摘要]
</source>

<rules>
1. 学术领域术语使用该领域的标准中文翻译
2. 如果某术语没有公认的中文翻译，保留英文
3. 保持学术论文的正式语气
4. 保留所有引用标记
5. 关键数据和结论要精确翻译，不能概括
</rules>
```

## 数据分析场景

### SQL 生成

```xml
<task>
基于以下数据库 schema，生成 SQL 查询
</task>

<schema>
users (id, name, email, created_at, plan_type)
orders (id, user_id, amount, status, created_at)
products (id, name, category, price)
order_items (id, order_id, product_id, quantity)
</schema>

<question>
找出过去 90 天内，消费金额排名前 10 的用户，
包括他们的姓名、总消费金额、订单数量、最常购买的产品类别。
</question>

<constraints>
- 使用 PostgreSQL 语法
- 只包含已完成的订单（status = 'completed'）
- 金额保留两位小数
- 添加注释解释每个 JOIN 和子查询的目的
</constraints>
```

### 数据解读

```
以下是上月的用户数据：

<data>

|------|------|------|------|
| DAU | 12,500 | 15,000 | -16.7% |
| 新注册 | 3,200 | 2,800 | +14.3% |
| 付费转化 | 2.1% | 3.5% | -40% |
| 流失率 | 8.2% | 5.1% | +60.8% |
| ARPU | $12.5 | $14.2 | -12.0% |
</data>

请分析：
1. 最值得关注的信号是什么？
2. 数据之间有什么关联？
3. 可能的原因假设（列 3 个）
4. 建议的下一步调查方向
5. 需要补充哪些数据来验证假设

用数据说话，不要空洞的建议。
```

## 通用优化原则

1. **给上下文**：Claude 了解背景后输出质量显著提升
2. **指定受众**：给技术总监的报告 ≠ 给实习生的教程
3. **限制长度**：没有长度限制，Claude 会过度扩展
4. **要求格式**：表格、列表、标题——结构化比长段文字更有效
5. **迭代优化**：第一版不满意就给具体反馈让它改，而不是重新写 prompt
