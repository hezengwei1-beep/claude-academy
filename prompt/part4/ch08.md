
# 第八章 CLAUDE.md 与项目级 Prompt 管理

## CLAUDE.md 就是项目级的 System Prompt

在 Claude Code 中，CLAUDE.md 文件的内容会被自动注入到每次会话的上下文中。本质上它就是一个**持久化的、项目绑定的 System Prompt**。

## 三级层次

```
~/.claude/CLAUDE.md          → 全局（你的个人偏好）
~/Projects/my-app/CLAUDE.md  → 项目级（项目约定）
~/Projects/my-app/src/CLAUDE.md  → 目录级（模块特殊规则）
```

三级会叠加：Claude 同时读取所有匹配的 CLAUDE.md。

## 2,500 Token 法则

### 为什么要精简

在 Agent Teams 中，每个 teammate 都会读取 CLAUDE.md：
- 5 个 teammates × 5,000 tokens CLAUDE.md = 25,000 tokens 浪费
- 5 个 teammates × 2,500 tokens CLAUDE.md = 12,500 tokens（省一半）

### 精简原则

**保留**：
- 不看代码无法推导的规则
- 项目的关键技术选型
- 容易犯的错误（gotchas）

**删除**：
- .eslintrc 中已有的代码规范
- package.json 中已有的依赖信息
- 目录结构（Claude 能自己看）
- 项目历史（git log 可查）

## 黄金模板

```markdown
## Stack
Next.js 14 App Router, TypeScript strict, Prisma, PostgreSQL

## Commands
- `pnpm dev` → dev server on :3000
- `pnpm test` → vitest
- `pnpm db:migrate` → prisma migrate dev

## Rules
- API routes: src/app/api/[resource]/route.ts
- Components: PascalCase, co-located tests
- No `any`, no `@ts-ignore`
- Prisma only in server components / API routes
- Auth via middleware.ts (edge runtime)

## Gotchas
- Rate limiter in Redis, not memory
- File uploads → S3, not local
- Env vars: .env.local for dev, Vercel for prod
```

~800 tokens，覆盖了 Claude 需要知道的一切。

这个 CLAUDE.md 可能有 5,000+ tokens，其中 80% 是冗余信息。

## 全局 CLAUDE.md 最佳实践

`~/.claude/CLAUDE.md` 放你的**个人偏好**，适用于所有项目：

```markdown
- 中文沟通，代码/日志可英文
- 回答简洁，结论先行
- 复杂任务先给方案让我确认
- 禁止未经确认的 git push
- 禁止 --force 推送
```

## Agent Teams 专属指令

如果你经常用 Agent Teams，可以在项目 CLAUDE.md 中加入：

```markdown
## Agent Teams 规则
- 修改文件前确认在你的职责范围内
- 接口变更先发消息给相关 teammate
- 不确定的设计决策发消息给 Lead
- 不要用 `any` 类型绕过类型检查
```

## 维护节奏

| 时机 | 操作 |
|------|------|
| 项目初期 | 频繁更新（每周审查） |
| 项目稳定后 | 偶尔修正（每月审查） |
| 新技术引入 | 立即更新 |
| 新人吐槽 Claude 行为 | 检查是否需要加规则 |

CLAUDE.md 是活文档。过时的规则比没有规则更有害。
