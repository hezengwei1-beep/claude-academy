
# 第十章 API 参数调优与 Prompt 评估

## 关键 API 参数

### temperature

控制输出的随机性。范围 0-1。

| 值 | 行为 | 适用场景 |
|-----|------|---------|
| 0 | 完全确定性（相同输入→相同输出） | 数据提取、分类、代码生成 |
| 0.3 | 轻微随机 | 技术写作、分析报告 |
| 0.7 | 中等随机 | 创意写作、brainstorm |
| 1.0 | 最大随机 | 很少使用（输出可能不连贯） |

**经验法则**：
- 需要一致性 → temperature = 0
- 需要创意 → temperature = 0.5-0.7
- 默认 → 不设（Claude 会自动选择合适的值）

### top_p

控制 token 采样的范围。与 temperature 类似但机制不同。

**建议**：通常调 temperature 就够了，不需要同时调 top_p。

### max_tokens

限制输出的最大长度。

```python
# 短回答
response = client.messages.create(max_tokens=200, ...)

# 长文档
response = client.messages.create(max_tokens=4096, ...)

# 代码生成（可能较长）
response = client.messages.create(max_tokens=8192, ...)
```

**注意**：设太小可能导致输出被截断。设太大可能浪费钱（但不会影响质量）。

### stop_sequences

遇到指定字符串时停止生成：

```python
response = client.messages.create(
    stop_sequences=["```", "---"],  # 遇到代码块结束或分隔线就停
    ...
)
```

## Prompt 评估方法

### 方法一：人工评分

最简单但最有效。设计评分标准：

```
评分维度（每项 1-5 分）：
1. 准确性：信息是否正确
2. 完整性：是否覆盖了所有要点
3. 格式：是否符合要求的输出格式
4. 可用性：输出是否可以直接使用
5. 简洁性：是否没有冗余信息
```

### 方法二：A/B 测试

对比两个不同的 prompt，用相同的 10-20 个测试用例：

```python
test_cases = [
    {"input": "...", "expected": "..."},
    {"input": "...", "expected": "..."},
    # ...
]

results_a = [call_claude(prompt_a, tc["input"]) for tc in test_cases]
results_b = [call_claude(prompt_b, tc["input"]) for tc in test_cases]

# 人工评分对比
```

### 方法三：自动化评估

用 Claude 评估 Claude 的输出：

```python
evaluation_prompt = """
评估以下 AI 输出的质量：

<task>
{original_task}
</task>

<output>
{ai_output}
</output>

<criteria>
1. 准确性 (1-5)
2. 完整性 (1-5)
3. 格式符合度 (1-5)
</criteria>

输出 JSON: {"accuracy": N, "completeness": N, "format": N, "explanation": "..."}
"""
```

**注意**：AI 评估 AI 有偏见。只作为参考，关键决策仍需人工判断。

### 方法四：回归测试

维护一组"黄金测试用例"，每次修改 prompt 后自动验证：

```python
golden_tests = [
    {
        "input": "What is MCP?",
        "must_contain": ["Model Context Protocol", "AI", "连接"],
        "must_not_contain": ["我不知道", "抱歉"],
        "max_length": 500,
    },
]

def test_prompt(prompt, tests):
    for test in tests:
        output = call_claude(prompt, test["input"])
        for keyword in test["must_contain"]:
            assert keyword in output, f"Missing: {keyword}"
        for keyword in test["must_not_contain"]:
            assert keyword not in output, f"Unwanted: {keyword}"
        assert len(output) <= test["max_length"]
```

## Prompt 迭代流程

```
1. 写初版 prompt
   ↓
2. 用 5 个测试用例验证
   ↓
3. 分析失败案例
   ↓
4. 针对性修改 prompt
   ↓
5. 回归测试（确保新改不破坏旧的）
   ↓
6. 扩大测试（20 个用例）
   ↓
7. 稳定后部署
   ↓
8. 持续监控（定期抽查输出质量）
```

## 常见调优技巧

1. **加约束比加自由更有效**：明确说"不要做什么"比"你可以做任何事"好
2. **示例胜过描述**：一个好的 few-shot 示例 > 三段话的描述
3. **System prompt 不要太长**：2000 tokens 以内，关键信息浓缩
4. **测试边界情况**：空输入、超长输入、恶意输入
5. **版本管理 prompt**：把 prompt 当代码管理，git 跟踪变更
