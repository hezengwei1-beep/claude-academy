# 第一章 Claude 模型家族与选择指南

## 2026 年的 Claude 家族

| 模型 | 定位 | Input 价格 | Output 价格 | 上下文 | 最佳场景 |
|------|------|-----------|------------|--------|---------|
| **Opus 4.6** | 最强大脑 | $5/M | $25/M | 1M tokens | 复杂推理、架构设计、高质量写作 |
| **Sonnet 4.6** | 性价比之王 | $3/M | $15/M | 1M tokens | 日常编码、审查、分析 |
| **Haiku 4.5** | 速度优先 | ~$0.25/M | ~$1.25/M | 200K tokens | 分类、提取、简单问答 |

## 如何选择

```
任务需要深度推理或创意？
├── 是 → Opus 4.6
└── 否 → 任务需要速度和低成本？
    ├── 是 → Haiku 4.5
    └── 否 → Sonnet 4.6（默认选择）
```

### Opus 的优势场景
- 多步推理和复杂逻辑
- 高质量长文写作
- 架构设计和方案评估
- 模糊需求的理解和澄清
- Agent Teams 的 Team Lead

### Sonnet 的优势场景
- 日常代码生成和修改
- 代码审查和 Bug 修复
- 数据分析和报告
- 翻译和内容改写
- Agent Teams 的 Teammates

### Haiku 的优势场景
- 文本分类和标签
- 数据提取（从非结构化文本中提取结构化信息）
- 简单问答
- Subagent 的简单搜索任务
- 批量处理（成本敏感）

## Extended Thinking（深度思考）

Claude 4.6 系列支持 Extended Thinking——模型在回答前先进行内部推理。

### 什么是 Extended Thinking

```
普通模式：
  用户问题 → Claude 直接回答

Extended Thinking 模式：
  用户问题 → Claude 先在 <thinking> 中推理 → 然后给出回答
```

thinking 过程中 Claude 会：
- 分解问题
- 考虑多种可能性
- 自我纠正
- 组织逻辑

### 何时使用

| 场景 | 使用 Thinking？ |
|------|----------------|
| 数学推理 | 强烈建议 |
| 代码 Bug 定位 | 建议 |
| 架构设计 | 建议 |
| 写作 | 可选 |
| 简单问答 | 不需要 |
| 数据提取 | 不需要 |

### 成本注意

Thinking tokens 按 **output** 价格计费。一个需要大量思考的问题可能产生数千个 thinking tokens，成本显著高于普通回答。

## 上下文窗口的有效利用

1M tokens 听起来很多，但在以下场景中会很快用完：

- 大型代码库的多文件读取
- 长对话历史累积
- Agent Teams 中每个 teammate 的独立上下文

### 策略

1. **重要信息放前面**：Claude 对上下文开头和最近的内容关注度最高
2. **System prompt 精简**：不要把 10,000 tokens 的指令放在 system prompt
3. **长对话定期总结**：对话超过一定长度后，让 Claude 总结之前的内容
4. **分段处理**：大文件分段输入，而不是一次性塞入

## Claude 与 GPT-4 的 Prompt 差异

了解差异可以帮你更好地利用 Claude 的优势：

| 维度 | Claude | GPT-4 |
|------|--------|-------|
| XML 标签 | 响应极好，推荐使用 | 支持但无特别优势 |
| System Prompt 遵循度 | 极高 | 高 |
| 拒绝率 | 较高（更谨慎） | 较低 |
| 长输出质量 | 一致性高 | 容易退化 |
| 代码生成风格 | 偏保守、注重安全 | 偏激进、追求简洁 |
| 深度推理 | Extended Thinking 优秀 | Chain of thought |

**核心差异**：Claude 更像一个"谨慎的专家"，GPT-4 更像一个"灵活的通才"。
