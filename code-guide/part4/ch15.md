# 第二章 Keybindings、Memory 与 Status Line

## Keybindings 自定义

### 配置文件

`~/.claude/keybindings.json`：

```json
[
  {
    "key": "ctrl+r",
    "command": "/review",
    "description": "快速代码审查"
  },
  {
    "key": "ctrl+t",
    "command": "/test $CURRENT_FILE",
    "description": "为当前文件生成测试"
  },
  {
    "key": "ctrl+shift+c",
    "command": "/commit",
    "description": "智能提交"
  }
]
```

### Chord Bindings（组合键）

```json
[
  {
    "key": "ctrl+k ctrl+r",
    "command": "/review",
    "description": "Chord: 审查"
  },
  {
    "key": "ctrl+k ctrl+t",
    "command": "/test",
    "description": "Chord: 测试"
  }
]
```

先按 `ctrl+k`，再按 `ctrl+r`。

### 内置可覆盖的键

| 默认键 | 默认作用 | 可以重绑为 |
|--------|---------|-----------|
| `Enter` | 发送 | 保持不变 |
| `Shift+Enter` | 换行 | 保持不变 |
| `Escape` | 取消 | 自定义命令 |

## Memory 系统

### 什么是 Memory

Claude Code 有一个自动记忆系统。它会在 `~/.claude/projects/` 下为每个项目维护记忆文件，记住跨会话的信息。

### Memory 的类型

| 类型 | 触发 | 用途 |
|------|------|------|
| **user** | Claude 了解到你的信息 | 角色、偏好、技术背景 |
| **feedback** | 你纠正或确认 Claude 的行为 | 行为指导 |
| **project** | 了解到项目的非代码信息 | 截止日期、负责人、决策背景 |
| **reference** | 了解到外部资源 | 链接、工具、文档位置 |

### 手动控制

```
记住：我们的 staging 环境 URL 是 staging.example.com
```

```
记住：部署前必须先跑 integration test，上次跳过导致了线上事故
```

```
忘记：关于旧的 API endpoint 的记忆
```

### Memory 文件结构

```
~/.claude/projects/-Users-你的用户名-Projects-my-app/memory/
├── MEMORY.md            → 索引文件
├── user_role.md         → 用户信息
├── feedback_testing.md  → 行为反馈
└── project_deploy.md    → 项目信息
```

### 最佳实践

**该记的**：
- 你的角色和技术背景
- 项目的非显而易见的约定
- 之前踩过的坑
- 外部工具和资源的位置

**不该记的**：
- 代码结构（Claude 能自己看）
- Git 历史（git log 能查）
- 临时调试信息

## Status Line 配置

Claude Code 底部的状态栏可以自定义显示内容。

通过 settings.json 配置：

```json
{
  "statusLine": {
    "show": true,
    "items": ["model", "tokens", "cost"]
  }
}
```

显示当前模型、token 用量、成本估算。
