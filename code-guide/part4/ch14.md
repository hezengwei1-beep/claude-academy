# 第一章 自定义 Slash Commands

## 什么是 Slash Commands

Slash Commands 是你自定义的 `/` 命令。定义一次，以后在任何项目中一键触发。

## 创建方式

### 项目级 Slash Command

在项目根目录创建 `.claude/commands/` 目录：

```
.claude/commands/
├── review.md        → /review
├── test-gen.md      → /test-gen
└── daily-report.md  → /daily-report
```

每个 `.md` 文件就是一个命令，文件名就是命令名。

### 全局 Slash Command

放在 `~/.claude/commands/` 下，所有项目都可用。

## 文件格式

`review.md` 示例：

```markdown
审查当前的 git diff，按以下维度分析：

1. **安全性**：是否有 OWASP Top 10 漏洞
2. **性能**：是否有 N+1 查询或不必要的计算
3. **逻辑**：是否有边界情况未处理
4. **测试**：改动是否有测试覆盖

输出格式：
[Critical/High/Medium/Low] 文件:行号 - 问题描述 → 建议

最后给出整体评分（1-10）和一句话总结。
```

### 使用

在 Claude Code 中输入：

```
/review
```

Claude 会读取 `review.md` 的内容作为指令执行。

## 带参数的 Slash Command

使用 `$ARGUMENTS` 占位符：

`fix-issue.md`：

```markdown
修复 Issue #$ARGUMENTS：

1. 先查看 Issue 描述理解问题
2. 在代码中定位相关文件
3. 分析根本原因
4. 实现修复
5. 编写测试验证修复
6. 生成 commit message
```

使用：

```
/fix-issue 123
```

`$ARGUMENTS` 会被替换为 `123`。

## 10 个实用 Slash Commands

### /review — 代码审查

```markdown
审查 git diff 的改动。
维度：安全、性能、逻辑、测试。
格式：[严重度] 文件:行号 - 问题 → 建议
最后给评分和一句话总结。
```

### /commit — 智能提交

```markdown
查看所有未暂存和已暂存的改动。
生成一个简洁的中文 commit message，描述"为什么"而不是"是什么"。
先展示 message 让我确认，确认后执行 git commit。
```

### /test — 生成测试

```markdown
为 $ARGUMENTS 文件生成单元测试。
使用 vitest 框架。
覆盖：正常路径、错误路径、边界情况。
测试文件放在源文件同目录，命名 *.test.ts。
```

### /explain — 代码解释

```markdown
用简单的语言解释 $ARGUMENTS。
结构：一句话概述 → 逐步逻辑 → 设计决策 → 潜在改进。
假设读者了解基本编程但不了解这个项目。
```

### /refactor — 重构建议

```markdown
分析 $ARGUMENTS 的代码异味。
参考 SOLID、DRY、KISS 原则。
每个建议含：异味描述、为什么是问题、重构方案（含代码）、预期收益。
先给方案让我确认，不要直接改。
```

### /doc — 生成文档

```markdown
为 $ARGUMENTS 生成文档。
包含：概述、参数说明、返回值、使用示例、注意事项。
格式：JSDoc/TSDoc 注释 + README 段落。
```

### /perf — 性能分析

```markdown
分析 $ARGUMENTS 的性能问题。
关注：数据库查询、内存使用、算法复杂度、缓存机会。
给出具体的优化方案和预期提升。
```

### /security — 安全检查

```markdown
对 $ARGUMENTS 做安全审查。
关注 OWASP Top 10：注入、认证、敏感数据、XXE、访问控制。
每个发现含：漏洞类型、严重度、位置、修复方案。
```

### /migration — 迁移检查

```markdown
检查最近的数据库迁移文件：
1. 是否可回滚
2. 是否有数据丢失风险
3. 大表操作是否会锁表
4. 是否需要分步执行
给出安全执行建议。
```

### /standup — 每日站会

```markdown
基于昨天到现在的 git log，生成站会报告：
- 昨天完成：
- 今天计划：
- 阻塞：
格式简洁，每项一行。
```

## 团队共享

把 `.claude/commands/` 提交到 Git，团队成员 clone 后自动获得所有自定义命令。

```
.claude/commands/
├── review.md        → 团队统一的审查标准
├── commit.md        → 团队统一的提交规范
└── test.md          → 团队统一的测试模板
```
