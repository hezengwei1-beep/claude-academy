# 第一章 安装与首次运行

## 安装前检查

### 系统要求

| 条件 | 要求 |
|------|------|
| 操作系统 | macOS 14+ / Linux / Windows (WSL) |
| Node.js | >= 18（推荐 20 LTS） |
| 磁盘空间 | ~200MB |
| 网络 | 需要访问 api.anthropic.com |

### 检查 Node.js

```bash
node --version   # 需要 v18 以上
npm --version    # 需要 v9 以上
```

如果没有安装 Node.js：

```bash
# macOS（推荐用 nvm）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
nvm install 20
nvm use 20

# 或直接下载：https://nodejs.org
```

## 安装 Claude Code

```bash
npm install -g @anthropic-ai/claude-code
```

### 验证安装

```bash
claude --version
# 输出类似：2.1.119 (Claude Code)
```

### 常见安装问题

**问题：权限错误 `EACCES`**
```bash
# 方案 1：修复 npm 权限
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc
source ~/.zshrc
npm install -g @anthropic-ai/claude-code

# 方案 2：使用 sudo（不推荐但有效）
sudo npm install -g @anthropic-ai/claude-code
```

**问题：`claude` 命令找不到**
```bash
# 检查全局 bin 路径是否在 PATH 中
npm root -g   # 查看全局安装路径
echo $PATH    # 检查 PATH

# 添加到 PATH（zsh）
echo 'export PATH="$(npm root -g)/../bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

**问题：Node.js 版本太低**
```bash
nvm install 20 && nvm use 20
```

### 更新

```bash
npm update -g @anthropic-ai/claude-code

# 验证更新成功
claude --version
```

## 首次运行

### Step 1：进入项目目录

```bash
cd ~/Projects/my-project   # 你的任何代码项目
```

> 重要：Claude Code 在**项目目录**中运行效果最好。它会自动识别项目的语言、框架、文件结构。

### Step 2：启动

```bash
claude
```

### Step 3：首次认证

首次启动会引导你登录。有两种方式：

**方式 A：浏览器登录（推荐）**
1. Claude Code 会打开浏览器
2. 用你的 Anthropic 账号登录
3. 授权 Claude Code
4. 自动返回终端

**方式 B：API Key**
```bash
export ANTHROPIC_API_KEY="sk-ant-api03-..."
claude
```

### Step 4：你看到的界面

```
╭─────────────────────────────────────────╮
│  Claude Code v2.1.119                    │
│                                          │
│  Model: claude-sonnet-4-6               │
│  Context: 1,000,000 tokens              │
│  Project: ~/Projects/my-project          │
│                                          │
│  Type a message or /help for commands    │
╰─────────────────────────────────────────╯

>
```

看到 `>` 提示符，你就可以开始对话了。

## 核心概念

### Claude Code 是什么

Claude Code 是一个**终端中的 AI 编码代理**。注意"代理"二字——它不只是回答问题，它能**直接动手**：

| 能力 | 说明 | 示例 |
|------|------|------|
| 读取文件 | 自动读取项目文件 | 读 README 理解项目 |
| 修改代码 | 直接编辑源文件 | 修复 bug、重构 |
| 运行命令 | 执行 shell 命令 | npm test、git status |
| 搜索代码 | 跨文件搜索 | 找到所有调用某函数的地方 |
| Git 操作 | 版本控制 | commit、diff、log |
| 创建文件 | 生成新文件 | 写测试、生成配置 |

### 与其他 AI 工具的对比

| 工具 | 定位 | Claude Code 的区别 |
|------|------|-------------------|
| ChatGPT | 通用聊天 | Claude Code 直接操作你的代码 |
| GitHub Copilot | IDE 代码补全 | Claude Code 是完整的代理（理解+执行） |
| Cursor | IDE AI 助手 | Claude Code 独立于 IDE，更灵活 |

### 心智模型

把 Claude Code 想象成一个**远程的资深同事**：
- 你告诉他"这里有个 bug"
- 他自己去看代码、分析原因、提出修复方案
- 你确认后他直接改
- 你不需要告诉他"先打开某个文件、看第几行"——他会自己找

## 第一个任务

### 了解项目

在项目目录中启动 Claude Code，输入：

```
这个项目是做什么的？给我一个简要概述。
```

**Claude 会做什么**：
1. 读取 README.md
2. 读取 package.json / pyproject.toml 等配置
3. 浏览目录结构
4. 读取几个关键源文件
5. 综合给你一个项目概述

**你不需要告诉它去读哪些文件**——它会自己判断。

### 十个新手友好的任务

从简单到复杂，试试这些：

```
1. 这个项目用了什么技术栈？

2. 找出所有 TODO 注释

3. 最近三次 commit 改了什么？

4. src/auth/ 目录的代码做了什么？解释主要逻辑。

5. package.json 里有没有过期的依赖？

6. 运行测试，告诉我结果。

7. 找出代码中所有硬编码的字符串常量。

8. 这个项目的 .env.example 有哪些必填的环境变量？

9. 帮我写一个 .gitignore，覆盖这个项目类型常见的忽略文件。

10. 分析这个项目的目录结构，画一个架构概览图（用文字描述）。
```

前 5 个是只读操作（零风险），后 5 个可能涉及文件操作（Claude 会在执行前请求确认）。

## 三种工作模式详解

### 模式一：交互模式（默认）

```bash
claude
```

启动后进入对话界面。这是最常用的模式。

**适合**：日常开发、探索性任务、需要多轮对话的复杂任务

**对话示例**：
```
> 找出 src/auth/ 中可能的安全问题

Claude: 我来检查 src/auth/ 目录...
[读取 5 个文件]
发现 3 个潜在问题：
1. middleware.ts:42 — JWT secret 是硬编码的
2. login.ts:28 — 没有 rate limiting
3. token.ts:15 — refresh token 没有绑定 IP

> 帮我修复第一个问题

Claude: 我建议将 JWT secret 移到环境变量...
[展示修改方案]
确认修改？(y/n)

> y

Claude: 已修改 middleware.ts，将 JWT_SECRET 改为从 process.env 读取。
同时更新了 .env.example 添加了 JWT_SECRET 条目。
```

### 模式二：非交互模式（-p）

```bash
claude -p "列出 src/ 下所有 .ts 文件的行数统计"
```

执行一个任务然后退出。输出直接打印到 stdout。

**适合**：脚本、CI/CD、一次性查询

**实际例子**：
```bash
# 快速统计
claude -p "这个项目有多少行代码？按语言分组统计"

# 生成内容
claude -p "基于 src/api/ 的代码生成 API 文档" > docs/api.md

# JSON 输出（方便程序解析）
claude -p "列出所有 API 端点" --output json > endpoints.json
```

### 模式三：Pipe 模式

把其他命令的输出管道传给 Claude：

```bash
# 审查 Git 改动
git diff | claude -p "审查这些改动，找出潜在问题"

# 分析错误日志
cat error.log | claude -p "分析这个错误日志，找出根因"

# 分析测试失败
npm test 2>&1 | claude -p "分析哪些测试失败了，为什么"

# 审查 PR
gh pr diff 142 | claude -p "审查这个 PR 的改动"
```

**组合威力**：Pipe 模式让 Claude Code 成为 Unix 工具链的一部分。

## 权限模型

### 默认权限

| 操作类型 | 默认行为 | 风险级别 |
|----------|---------|---------|
| 读取文件 | ✅ 自动允许 | 无 |
| 搜索代码（Grep/Glob） | ✅ 自动允许 | 无 |
| 修改文件（Write/Edit） | ⚠️ 需要确认 | 中 |
| 运行命令（Bash） | ⚠️ 需要确认 | 中-高 |
| Git 操作 | ⚠️ 需要确认 | 中 |

### 确认界面

当 Claude 想修改文件时，你会看到：

```
Claude wants to edit src/auth/middleware.ts

 - const JWT_SECRET = "hardcoded-secret";
 + const JWT_SECRET = process.env.JWT_SECRET;

[Allow] [Allow Always] [Deny]
```

- **Allow**：本次允许
- **Allow Always**：以后同类操作都自动允许（写入 settings）
- **Deny**：拒绝

### 安全建议

刚开始使用时，建议**逐次确认**所有写操作。等你熟悉 Claude Code 的行为模式后，再逐步放开权限（详见 Part 3 权限章节）。
