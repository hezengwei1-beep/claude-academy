# 第八章 Skills 与 Plugins 生态

## 为什么需要 Skills

如果你发现自己每次使用 Cowork 都在重复输入类似的提示词——"按这个格式整理"、"用这个模板输出"、"先做 A 再做 B 再做 C"——那你需要的就是 Skills。

**Skills = 可复用的提示词模板 + 预定义的工作流程**

把它想象成 Excel 的宏，或者手机的快捷指令（Shortcuts）：你定义一次，以后一键触发。

## Skills 的基本概念

一个 Skill 包含：

1. **名称**：简短的标识（如"周报生成"、"发票提取"）
2. **描述**：这个 Skill 做什么（一句话）
3. **指令**：详细的步骤和约束（就是你平时写的提示词）
4. **输入参数**（可选）：每次运行时可能变化的部分
5. **输出格式**（可选）：预期的输出结构

## 创建你的第一个 Skill

### 示例：周报生成 Skill

**步骤**：
1. 在 Cowork 设置中找到 Skills 管理
2. 点击"创建新 Skill"
3. 填写信息：

```
名称：周报生成
描述：基于本周笔记自动生成周报

指令：
1. 读取 ~/Claude Cowork/INPUTS/本周笔记/ 下的所有文件
2. 按时间顺序梳理本周工作内容
3. 使用 ~/Claude Cowork/TEMPLATES/周报模板.md 中的模板格式
4. 填充以下模块：
   - 本周完成：列出完成的工作项
   - 进行中：列出仍在进行的工作
   - 下周计划：基于进行中的工作推导
   - 风险和阻塞：标注需要关注的问题
5. 量化成果（如有数据支撑）
6. 输出到 ~/Claude Cowork/OUTPUTS/周报-{当前日期}.md
```

### 使用方式

创建完成后，你只需要：
1. 在 Cowork 中输入"运行周报生成"
2. 或者通过手机 Dispatch 发送"周报生成"

Claude 会按照 Skill 中定义的步骤自动执行。

## Plugins 与 Connectors

除了 Skills（你自己定义的工作流），Cowork 还支持 Plugins 和 Connectors——连接外部服务的能力。

### Connectors 的作用

Connectors 让 Cowork 能够与外部服务交互：
- **Google Drive**：读取和写入 Google Docs、Sheets
- **Notion**：查询和更新 Notion 数据库
- 更多 Connectors 持续增加中

### 使用方式

在 Settings > Cowork > Connectors 中，你可以连接支持的外部服务。连接后，Cowork 可以在任务中直接访问这些服务。

**示例**：
```
从我的 Google Drive 中找到上周的会议记录，
整理成行动项，
然后更新到 Notion 的项目跟踪数据库中。
```

## Skills + Global Instructions 的配合

Skills 和 Global Instructions 是互补的：

| | Global Instructions | Skills |
|---|---|---|
| 作用范围 | 所有对话 | 特定任务 |
| 内容类型 | 你是谁、偏好是什么 | 具体任务怎么做 |
| 修改频率 | 很少改 | 经常增加新的 |

**最佳实践**：
- Global Instructions 放不变的东西：你的角色、语气偏好、文件命名规范
- Skills 放具体的工作流：周报怎么写、发票怎么提取、邮件怎么起草

两者结合，Claude 既知道"你是谁"，又知道"这个任务怎么做"，效率最大化。

## 进阶：Skill 的迭代优化

创建 Skill 后不要期望一步到位。建议的迭代流程：

1. **V1**：写一个基础版，跑一次看效果
2. **发现问题**：输出格式不对？遗漏了某个步骤？
3. **V2**：补充约束和步骤
4. **稳定后**：把这个 Skill 加入日常工作流

好的 Skill 是迭代出来的，不是一次写好的。
