# 第七章 Computer Use：让 Claude 操作你的电脑

## 从文件操作到屏幕操作

前面几章中，Cowork 操作的对象是**文件**——读取、移动、创建、修改文件。Computer Use 把能力范围扩大到了**屏幕**——Claude 可以看到你的屏幕，移动鼠标，点击按钮，输入文字。

简单来说：

| 能力 | 普通 Cowork | Computer Use |
|------|-------------|-------------|
| 读写文件 | ✅ | ✅ |
| 操作命令行 | ✅ | ✅ |
| 看到屏幕 | ❌ | ✅ |
| 点击按钮 | ❌ | ✅ |
| 输入表单 | ❌ | ✅ |
| 操作 GUI 应用 | ❌ | ✅ |

## Computer Use 的工作原理

Computer Use 的核心机制：

1. **截屏**：Claude 定期对你的屏幕截图
2. **理解**：通过视觉分析理解屏幕上的内容（按钮位置、文字、布局）
3. **操作**：发送鼠标移动、点击、键盘输入等指令
4. **验证**：再次截屏确认操作结果
5. **循环**：重复上述过程直到任务完成

这就像一个远程桌面操作——Claude "看到"你的屏幕并"动手"操作。

## 启用方式

在 Cowork 中，当你描述的任务需要 GUI 操作时，Claude 会提示你启用 Computer Use。

你也可以在 Settings > Cowork > Computer Use 中预先配置权限。

**权限控制**：
- 你可以选择 Claude 能操作的屏幕区域
- 你可以随时中断 Claude 的操作
- 敏感操作（如输入密码）需要你手动完成

## 适用场景

### 场景一：填写在线表单

```
帮我打开 [网站]，用以下信息填写注册表单：
- 姓名：...
- 邮箱：...
- 公司：...
不要提交，填完后等我确认。
```

### 场景二：操作不支持 API 的桌面应用

```
打开 Keynote，新建一个演示文稿：
- 使用"简约"主题
- 第一页标题：[你的标题]
- 第二页：要点列表，内容如下...
保存到 ~/Documents/演示/
```

### 场景三：跨应用的工作流

```
从 Numbers 表格中复制销售数据，
打开 Pages 文档模板，
把数据粘贴到报告的指定位置，
导出为 PDF。
```

### 场景四：网页信息采集

```
打开浏览器，访问 [网址]，
从页面上提取产品名称、价格和描述，
整理成表格保存到 ~/Claude Cowork/OUTPUTS/。
```

## 不适用的场景

### 涉及凭证的操作

不要让 Claude 操作涉及密码输入的页面。虽然 Claude 不会存储你的密码，但屏幕截图可能包含敏感信息。

**正确做法**：你手动登录后，再让 Claude 接管后续操作。

### 高精度的视觉操作

Computer Use 依赖截图识别，精度有限。以下任务可能不可靠：
- 精密的图片编辑
- 像素级的 UI 调整
- 快速动画或视频操作

### 需要实时反馈的操作

由于截屏-分析-操作的循环有延迟，实时性要求高的操作（如游戏、视频编辑时间线拖动）不适合。

## 安全边界

### Claude 不会做的事

- 不会在你不知情的情况下访问网站
- 不会主动打开你没要求的应用
- 不会存储截屏内容
- 不会输入它"猜测"的密码

### 你应该做的事

1. **不要在敏感页面启用 Computer Use**：银行网站、密码管理器等
2. **关闭不相关的窗口**：减少 Claude 看到无关信息的可能
3. **保持在旁边观察**：至少在首次使用时
4. **设置操作边界**：告诉 Claude 只在特定应用中操作

### 权限分级建议

| 场景 | 建议权限 |
|------|----------|
| 文件整理 | 普通 Cowork 即可，不需要 Computer Use |
| 填表单 | 允许 Computer Use，你在旁监督 |
| 跨应用操作 | 允许 Computer Use，明确指定应用范围 |
| 涉及登录 | 你手动登录，然后允许 Computer Use |
| 涉及支付 | 不建议使用 Computer Use |

## 实操建议

1. **先用小任务试水**：不要第一次就让 Claude 操作复杂的多应用工作流
2. **给明确的步骤**：Computer Use 在有清晰步骤时表现更好
3. **留观察窗口**：让你能看到 Claude 在做什么
4. **设置"暂停点"**：在关键步骤让 Claude 暂停等你确认

## 实操演示：完整的 Computer Use 任务

### 任务：从 Numbers 提取数据生成报告

```
请帮我完成以下操作：

1. 打开 ~/Documents/销售数据.numbers
2. 从"月度汇总"工作表中读取 2026 年 Q1 的数据
3. 复制销售额和用户数两列
4. 打开 Pages，新建文档
5. 用"商务报告"模板
6. 在标题写"2026 Q1 销售报告"
7. 粘贴数据表格
8. 在下方写一段 100 字的分析摘要
9. 导出为 PDF 到 ~/Documents/Reports/
10. 不要关闭任何应用

每一步操作前暂停让我确认。
```

### Claude 的执行过程

```
Step 1: 打开 Numbers...
[截屏] 识别到 Numbers 图标，双击打开
[截屏] 文件打开成功，看到工作表列表

准备进行 Step 2，是否继续？
> 继续

Step 2: 切换到"月度汇总"工作表...
[截屏] 点击底部的"月度汇总"标签
[截屏] 看到 Q1 数据：1月/2月/3月 的销售额和用户数

准备复制数据，是否继续？
> 继续

Step 3: 选中数据区域...
[截屏] 选中 A1:C4（标题行 + 3 个月数据）
[截屏] Cmd+C 复制
...
```

每一步 Claude 都：
1. 截屏确认当前状态
2. 执行一个操作（点击/输入/快捷键）
3. 再次截屏确认操作结果
4. 等待你确认后继续

### 成功率和失败处理

**高成功率的操作**：
- 点击大按钮和菜单项
- 输入文字
- 使用快捷键（Cmd+C, Cmd+V 等）
- 切换窗口和标签

**可能失败的操作**：
- 点击小按钮（<20px）
- 拖拽操作
- 需要精确坐标的操作
- 动画过程中的操作

**失败时怎么办**：
Claude 会识别到操作没有达到预期效果，自动重试或询问你如何继续。如果你发现它点错了位置，按 `Ctrl+C` 中断然后手动操作那一步。

## Computer Use + 普通 Cowork 的组合

最实用的模式不是纯 Computer Use，而是**混合使用**：

```
1. 用普通 Cowork 分析数据（文件操作，快速准确）
2. 用 Computer Use 在 Keynote 中创建演示（需要 GUI）
3. 用普通 Cowork 保存最终文件（文件操作）
```

能用文件操作完成的部分就不用 Computer Use——文件操作更快更可靠。Computer Use 只用在**必须操作 GUI 的环节**。

## Computer Use 的演进方向

目前的 Computer Use 是"截屏-分析-操作"的循环模式，有延迟和精度限制。未来的演进方向：

1. **更高精度**：更好的 UI 元素识别
2. **更低延迟**：减少截屏-分析的循环时间
3. **更多应用支持**：原生集成更多桌面应用的 API
4. **录制回放**：录制一次操作，以后自动回放

Computer Use 是 Cowork 中最前沿的功能，还在持续迭代中。随着技术进步，它的精度和可靠性会不断提升。
