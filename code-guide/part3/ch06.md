# 第五章 十大日常场景实战

本章精选十个最常用的 Cowork 场景，每个场景都包含：问题描述、操作步骤、提示词模板和预期结果。

## 场景一：文件智能整理

**问题**：Downloads 文件夹里堆了几百个文件，手动分类要半天。

**提示词模板**：
```
帮我智能整理 ~/Downloads 文件夹：
1. 按语义分类（不只是扩展名）：
   - 发票和收据
   - 技术文档和电子书
   - 图片和截图
   - 安装包和压缩包
   - 其他
2. 对 PDF 文件，读取前几页内容判断它是发票、电子书还是论文
3. 创建对应的子文件夹并移动文件
4. 生成整理报告到 ~/Claude Cowork/OUTPUTS/整理报告.md
5. 不要删除任何文件
```

**亮点**：Cowork 不只看文件扩展名，还能读取 PDF 内容做语义分类。一张发票和一本电子书虽然都是 PDF，但会被分到不同类别。

**预期耗时**：3-10 分钟（取决于文件数量）

## 场景二：多源笔记合成报告

**问题**：散落在多个文件中的会议笔记、调研记录、想法片段，需要合成一份结构化报告。

**提示词模板**：
```
我在 ~/Claude Cowork/INPUTS/项目笔记/ 下有多个笔记文件。请：
1. 读取所有文件，理解每个文件的核心内容
2. 识别共同主题和关键发现
3. 合成一份结构化报告，包含：
   - 执行摘要（200 字以内）
   - 关键发现（按主题分组）
   - 行动建议
   - 信息来源索引
4. 输出到 ~/Claude Cowork/OUTPUTS/综合报告.md
5. 格式：Markdown，使用标题、列表和表格
```

## 场景三：会议纪要处理

**问题**：有一份原始的会议记录（可能是语音转文字的粗稿），需要变成结构化的行动项文档。

**提示词模板**：
```
把 ~/Claude Cowork/INPUTS/会议记录-0425.txt 转化为结构化纪要：
1. 会议基本信息（日期、参会者、主题）
2. 讨论要点摘要（每个议题 2-3 句话）
3. 决策事项（明确列出做了什么决定）
4. 行动项表格：
   | 事项 | 负责人 | 截止日期 | 优先级 |
5. 开放问题（待讨论/待决策的事项）
6. 输出到 ~/Claude Cowork/OUTPUTS/
```

## 场景四：竞品研究与分析

**问题**：需要快速了解某个竞品的基本情况。

**提示词模板**：
```
请帮我做一份 [竞品名称] 的研究报告：
1. 产品概况（一句话定位、核心功能）
2. 目标用户和定价
3. 与我们产品的功能对比表
4. 优势和劣势分析
5. 对我们的启示和建议

参考资料放在 ~/Claude Cowork/INPUTS/竞品资料/ 目录下。
如果资料不足，标注"需要补充"。
输出到 ~/Claude Cowork/OUTPUTS/竞品分析-[竞品名].md
```

## 场景五：数据提取与清洗

**问题**：有一批 PDF 发票或表格，需要提取关键信息并汇总。

**提示词模板**：
```
请从 ~/Claude Cowork/INPUTS/发票/ 目录下的所有 PDF 发票中提取：
1. 发票日期
2. 供应商名称
3. 金额（税前、税额、总额）
4. 发票编号

汇总为一个表格，输出为：
- Markdown 表格：~/Claude Cowork/OUTPUTS/发票汇总.md
- CSV 文件：~/Claude Cowork/OUTPUTS/发票汇总.csv

按日期排序，最后加一行合计。
```

## 场景六：邮件/消息批量起草

**问题**：需要给不同的人发类似但个性化的消息。

**提示词模板**：
```
我需要给以下人员发送项目进度更新邮件。

收件人信息在 ~/Claude Cowork/INPUTS/联系人.md 中。

要求：
1. 每封邮件个性化（提到对方关心的模块/领域）
2. 语气：专业但友好
3. 结构：项目整体进度 → 对方关心的部分 → 下一步计划 → 需要对方的配合
4. 每封邮件控制在 200 字以内
5. 所有邮件输出到 ~/Claude Cowork/OUTPUTS/邮件草稿/ 目录下，每封一个文件
```

## 场景七：简历/求职信定制

**问题**：针对不同的职位，需要定制简历和求职信。

**提示词模板**：
```
我的基础简历在 ~/Claude Cowork/INPUTS/简历-基础版.md。
目标职位的 JD 在 ~/Claude Cowork/INPUTS/JD-[公司名].md。

请：
1. 分析 JD 中的关键要求和偏好关键词
2. 基于我的经历，重新组织简历，突出与 JD 匹配的经验
3. 写一封 300 字以内的求职信
4. 列出我的经历与 JD 要求的匹配度分析表
5. 输出到 ~/Claude Cowork/OUTPUTS/求职-[公司名]/
```

## 场景八：学术论文辅助

**问题**：散乱的文献笔记需要整理，论文框架需要梳理。

**提示词模板**：
```
我在 ~/Claude Cowork/INPUTS/论文笔记/ 下有我的文献阅读笔记。

请帮我：
1. 整理所有文献笔记，按主题聚类
2. 识别文献之间的关系（支持/反对/补充）
3. 生成文献综述大纲（按逻辑线索组织，不是按文献逐个���列）
4. 标注每个主题下的核心争论和研究空白
5. 输出到 ~/Claude Cowork/OUTPUTS/文献综述框架.md
```

## 场景九：社交媒体内容生成

**问题**：需要一周的多平台社交媒体内容，手动写太慢。

**提示词模板**：
```
基于以下主题，帮我生成一周的社交媒体内容：

主题：[你的主题]
品牌调性：[专业/轻松/幽默]
目标受众：[描述]

每天生成：
- LinkedIn 帖子（专业版，200-300 字）
- X/Twitter 帖子（精简版，280 字符以内）
- 小红书笔记（中文，带 emoji，500 字以内）

要求：
- 周一到周五，每天一个小主题
- 内容之间有连贯性但各自独立
- 每条配一个 CTA（行动号召）
- 输出到 ~/Claude Cowork/OUTPUTS/本周内容/ 目录下
```

**参考成本**：社区用户报告，用 Cowork 生成一周多平台内容大约耗时 15 分钟，成本约 $7.80。

## 场景十：周报/月报自动化

**问题**：每周都要写周报，格式相同但内容不同，重复劳动。

**提示词模板**：
```
请帮我生成本周周报。

信息来源：
- ~/Claude Cowork/INPUTS/本周笔记/ 下的所有文件
- 如果有 git 仓库可以参考最近的 commit 记录

周报模板在 ~/Claude Cowork/TEMPLATES/周报模板.md

要求：
1. 按模板格式填充
2. 重点项目进展用 bullet points
3. 量化成果（如果有数据）
4. 下周计划基于本周未完成项推导
5. 输出到 ~/Claude Cowork/OUTPUTS/周报-2026-W17.md
```

## 使用技巧总结

1. **给足上下文**：告诉 Claude 你是谁、背景是什么、输出给谁看
2. **明确交付物**：指定输出格式、文件名、存放位置
3. **设定约束**：不要删除、不要修改原文件、字数限制等
4. **分步描述**：用编号列出步骤，比一大段话更清晰
5. **先审后执行**：检查 Claude 的执行计划再放行
