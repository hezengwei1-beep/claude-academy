
# 附录 H：各章小结与练习题

> 本附录汇总了所有章节的核心要点和动手练习。你可以作为独立的速查手册使用，也可以在读完每章后翻到这里做巩固。


## Ch1 AI 协作的时代背景

**核心要点**：
- AI 协作从"问答助手"演进到"可编排的团队"
- 单 agent 的瓶颈：上下文窗口、串行执行、缺乏专业化
- 多代理的三大价值：分工、并行、专业化

**练习**：
1. 列举你日常工作中 3 个天然可以并行化的任务
2. 估算这些任务如果用 3 个 agent 并行，能节省多少时间


## Ch2 Claude 协作体系全景

**核心要点**：
- 四层架构：Cowork → Agent Teams → Subagents → Worktrees
- 每层面向不同人群和场景
- Claude 的核心优势：本地执行 + 多代理通信

**练习**：
1. 画出四层架构的关系图（纸上手画即可）
2. 对于你当前的项目，哪一层最适合？为什么？


## Ch3 我该用哪个？决策树

**核心要点**：
- 三问决策法：需要通信？需要独立分支？需要隔离上下文？
- 3-5 个 agents 是甜区
- 不是所有任务都需要多 agent

**练习**：
1. 用决策树分析你最近的 5 个开发任务，看哪些适合多 agent
2. 估算使用 Agent Teams 的成本乘数


## Ch4 20 分钟上手 Cowork

**核心要点**：
- 前置条件：付费计划 + Apple Silicon Mac + Claude Desktop
- Global Instructions 是最重要的配置
- 工作文件夹要有清晰的结构

**练习**：
1. 完成 Cowork 的首次配置
2. 运行第一个文件整理任务
3. 检查五大常见错误，看自己是否有犯


## Ch5 十大日常场景实战

**核心要点**：
- 好的任务描述 = 目标 + 约束 + 交付物
- Cowork 能做语义理解（如区分 PDF 是发票还是电子书）
- 先审后执行

**练习**：
1. 从十大场景中选 3 个你用得到的，试运行一遍
2. 把效果最好的保存为 Skill


## Ch6 Dispatch：手机远程派任务

**核心要点**：
- 手机发任务 → 桌面执行 → 手机看结果
- 桌面必须保持唤醒
- 适合明确的、可自主完成的任务

**练习**：
1. 设置 Dispatch 并从手机发送一个简单任务
2. 测试电脑休眠后 Dispatch 的行为


## Ch7 Computer Use

**核心要点**：
- Claude 可以看屏幕、点鼠标、输文字
- 不适合涉及密码和支付的操作
- 精度有限，适合简单 GUI 操作

**练习**：
1. 让 Claude 打开一个应用并执行简单操作
2. 测试 Claude 在复杂 GUI 中的操作精度


## Ch8 Skills 与 Plugins 生态

**核心要点**：
- Skills = 可复用的提示词模板
- 好的 Skill 是迭代出来的
- Skills + Global Instructions 互补

**练习**：
1. 创建 3 个常用的 Skills
2. 运行每个 Skill 至少两次，优化提示词


## Ch9 架构解析：四组件模型

**核心要点**：
- 四组件：Team Lead、Teammates、Task List、Mailbox
- Mailbox 是区别于 Subagents 的核心
- 类比微服务架构

**练习**：
1. 用文字描述一个 3 人 Agent Teams 的数据流图
2. 列出五个架构局限性


## Ch10 环境配置与首次启动

**核心要点**：
- 需要 v2.1.32+，设置环境变量启用
- 两种显示模式：in-process 和 tmux
- 配置验证三步法

**练习**：
1. 完成环境配置
2. 创建一个 2 人最小团队验证功能正常


## Ch11 团队编排基础

**核心要点**：
- 好的 spawn 指令需要：名称、职责、文件范围、完成标准
- 任务生命周期：pending → in_progress → completed
- 快捷键操作熟练度决定效率

**练习**：
1. 创建 3 人审查团队审查一个 PR
2. 练习所有快捷键直到熟练


## Ch12 Delegate Mode

**核心要点**：
- Delegate Mode = Lead 不碰代码只做协调
- 保护 Lead 上下文是核心价值
- 3+ teammates 时强烈推荐

**练习**：
1. 用 Delegate Mode 协调一次 3 人任务
2. 对比 Delegate Mode 和默认模式的 Lead 上下文占用


## Ch13 Plan → Team 工作流

**核心要点**：
- 先想清楚再动手，避免方向错误的 5 倍浪费
- 好的计划 5-10 分钟产出
- 可选的 teammate 计划审批环节

**练习**：
1. 用 /plan 为一个中等任务制定计划
2. 按计划创建团队并执行


## Ch14 文件边界与冲突管理

**核心要点**：
- 一个文件同一时间只能有一个 teammate 修改
- spawn 时必须指定文件范围
- Delegate Mode + 文件边界 = 消除最常见失败

**练习**：
1. 为一个 3 人团队设计文件边界方案
2. 模拟一次文件冲突并练习解决


## Ch15 Subagents 子代理实战

**核心要点**：
- 层级关系，只向主 agent 汇报
- 四种创建方式：/agents、自然语言、@-mention、/fork
- 用 subagent 隔离探索型任务

**练习**：
1. 用 3 个 subagent 并行调查一个代码库
2. 测试不同模型（opus vs haiku）对 subagent 结果的影响


## Ch16 Git Worktrees 并行开发

**核心要点**：
- 完全独立的代码副本和分支
- `claude --worktree <name>` 快速创建
- 无改动时自动清理

**练习**：
1. 创建两个 worktree，分别做不同的任务
2. 测试 .worktreeinclude 文件的效果


## Ch17 选择指南

**核心要点**：
- 60 秒三问决策框架
- Subagents = 不需通信，Agent Teams = 需要通信
- 混合模式：可以组合使用

**练习**：
1. 用决策框架分析 5 个任务场景
2. 实践一次混合模式（Agent Teams + Subagents）


## Ch18 案例：16 代理构建 C 编译器

**核心要点**：
- 100K 行 Rust、$20K、2 周、99% 测试通过率
- 去中心化协调：文件锁 + Git 同步
- 关键策略：并行测试修复、GCC Oracle、专业化分工
- 实际加速比 4-8x（非理想的 16x）

**练习**：
1. 在你的项目中找一个可以用"Oracle 策略"的场景
2. 设计一个 5 agent 的分工方案处理你项目中最大的独立测试集


## Ch19 案例：多代理代码审查

**核心要点**：
- 审查率 16% → 54%，误报率 <1%
- 不批准 PR，审批是人的决定
- 每次审查 $15-25，20 分钟

**练习**：
1. 设计一个适合你团队的多代理审查流程
2. 估算引入自动审查后的 ROI


## Ch20 案例：企业级部署

**核心要点**：
- 成功企业的共性：高层支持、集成现有工作流、选择性使用、成本监控
- Claude Code 占工程师薪资 1-1.7%
- 非开发者也可以使用

**练习**：
1. 制定一个 3 个月的团队引入计划
2. 计算你团队的预估月度成本


## Ch21 案例：失败与教训

**核心要点**：
- 五条黄金法则：你是 PM、设 budget、测试≠完成、不要甩手、从小开始
- $47K 失控 = 无人看管 + 无 budget
- 49 agent = 规模过大 + 无文件边界

**练习**：
1. 检查你当前的 Claude Code 配置是否有 token budget
2. 列出你的使用场景中最可能出现的 3 个失败模式


## Ch22 成本控制与 Token 优化

**核心要点**：
- 七个优化策略（最重要的：不必要的就别用多 agent）
- CLAUDE.md 精简到 ~2,500 tokens
- Subagent 用 Haiku 模型

**练习**：
1. 测量你的 CLAUDE.md 的 token 数
2. 计算一次典型 Agent Teams 任务的成本


## Ch23 CLAUDE.md 最佳实践

**核心要点**：
- 只放"不看代码无法推导"的信息
- 三级作用域：全局 → 项目 → 目录
- Agent Teams 下每个 teammate 都读取，影响被放大

**练习**：
1. 将你的 CLAUDE.md 精简到 2,500 tokens 以内
2. 对比精简前后 Claude 的表现


## Ch24 Hooks 与质量门控

**核心要点**：
- 事件驱动：TaskCompleted → 自动跑测试/lint/安全扫描
- 保持轻量（30 秒以内）
- 幂等性和错误处理

**练习**：
1. 配置一个 TaskCompleted hook 自动运行测试
2. 测试 hook 失败时 Claude Code 的行为


## Ch25 社区工具生态

**核心要点**：
- 三层架构：内置 → 本地多 agent → 云端
- Claude Squad 是最大的社区开源工具
- 先掌握内置功能，按需引入第三方

**练习**：
1. 安装并试用 Claude Squad
2. 对比 Claude Squad 和 Agent Teams 的体验


## Ch26 反模式清单与排障指南

**核心要点**：
- 五大反模式：CLAUDE.md 过长、信任差距、无限探索、上下文遗忘、同文件冲突
- 通用排障流程：版本 → 配置 → 环境 → 网络 → 资源

**练习**：
1. 检查你的使用中是否存在五大反模式
2. 按排障流程解决一个你遇到的问题


## Ch27 Agent SDK 编程式构建

**核心要点**：
- SDK 适合可重复的自动化流程和产品集成
- 支持自定义工具
- Token budget 和超时控制是必须的

**练习**：
1. 用 SDK 写一个并行 2 个 subagent 的审查脚本
2. 为脚本添加 token budget 控制
