## R01: 智能 Git 提交

**场景**：每次提交都要想 commit message，烦。

**Slash Command** `.claude/commands/commit.md`：
```markdown
查看 git diff --staged（如果没有 staged 就看 git diff）。
生成一个中文 commit message：
- 一句话描述"为什么"改，不是"是什么"
- 如果改动涉及多个方面，用 bullet points
先展示给我确认，确认后执行 git add 和 git commit。
```

**使用**：`/commit`

## R04: 数据库迁移安全检查

`.claude/commands/migration-check.md`：
```markdown
检查最新的数据库迁移文件（migrations/ 或 prisma/migrations/）：

1. 是否可回滚？有没有 down migration？
2. 大表操作是否会锁表？（ALTER TABLE 加列/索引）
3. 是否有数据丢失风险？（DROP COLUMN 等）
4. 是否需要分步执行？
5. 生产环境执行预估时间？

输出：安全/警告/危险 + 详细说明 + 建议
```

## R05: 依赖更新分析

```markdown
分析 package.json 的依赖更新情况：
1. 运行 npm outdated
2. 对每个可更新的包：
   - 当前版本 vs 最新版本
   - 是 patch/minor/major？
   - major 更新的 breaking changes 摘要
3. 输出安全更新计划（先 patch → minor → 最后 major）
4. 标注哪些可以一起更新，哪些需要单独测试
```

## R16: Agent Teams 大型重构编排

```markdown
进入 Delegate Mode。

重构 src/legacy/ 为现代架构：

创建 4 人团队：
1. analyzer：分析现有代码的依赖关系和问题，输出重构计划
2. refactorer-a：负责 src/legacy/models/ → src/models/（Prisma）
3. refactorer-b：负责 src/legacy/controllers/ → src/api/（FastAPI 风格）
4. test-writer：为新代码编写测试

执行规则：
- analyzer 先完成，其他人再开始
- 每人只改自己的目录
- 接口变更先通知
```
