# 第五章 企业部署与团队治理

## 团队统一配置分发

### 标准项目模板

创建一个团队共享的 Claude Code 配置模板：

```
.claude/
├── commands/
│   ├── review.md         # 统一审查标准
│   ├── commit.md         # 统一提交规范
│   ├── test.md           # 统一测试模板
│   └── security.md       # 统一安全检查
├── settings.json         # 团队配置
└── settings.local.json   # .gitignore 中（个人覆盖）
```

### settings.json 团队模板

```json
{
  "model": "claude-sonnet-4-6",
  "permissions": {
    "allow": ["Read", "Glob", "Grep", "Bash(npm test)", "Bash(npm run lint)"],
    "deny": ["Bash(rm -rf *)", "Bash(git push --force)", "Bash(git reset --hard)"]
  },
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "command": "npx prettier --write $FILE_PATH 2>/dev/null || true"
      }
    ]
  }
}
```

## SSO 集成

企业版 Claude Code 支持 SSO：
- SAML 2.0
- OIDC
- 与现有的 Okta / Azure AD / Google Workspace 集成

管理员通过企业控制台管理：
- 哪些用户有权限使用
- 每个用户/团队的模型配额
- API Key 统一管理（用户不需要自己申请）

## 审计日志

企业版提供完整的审计追踪：

| 记录内容 | 说明 |
|----------|------|
| 工具调用 | 谁在什么时候调用了什么工具 |
| 文件操作 | 哪些文件被读取/修改 |
| 命令执行 | 执行了哪些 shell 命令 |
| MCP 调用 | 通过 MCP 访问了哪些外部系统 |
| Token 使用 | 每个用户的 token 消耗 |

## 成本治理

### 预算设置

```
每人每月上限：$300
每人每日上限：$30
单次会话上限：$50
```

### 监控告警

当用量接近阈值时自动通知：
- 80% 阈值：邮件通知用户
- 95% 阈值：通知管理员
- 100% 阈值：限制使用

### 成本报表

按团队/个人/项目维度生成月度成本报表：

```
团队 A：$2,340（15 人）
  ├── 人均：$156/月
  ├── 最高：张三 $280（重构任务）
  ├── 最低：李四 $45（轻度使用）
  └── 模型分布：Sonnet 78% / Opus 20% / Haiku 2%
```

### 优化建议

根据使用数据自动生成优化建议：
- "80% 的 Opus 调用可以降级为 Sonnet，预估月省 $400"
- "张三的平均会话长度是团队的 3 倍，建议使用 /compact"
- "项目 B 的 CLAUDE.md 有 8,000 tokens，建议精简"

## 安全策略

### 网络安全

- 所有通信 TLS 加密
- 支持 VPN/专线连接
- IP 白名单限制

### 数据安全

- 代码不离开本地（stdio MCP）
- API 请求加密传输
- 不持久化用户代码

### 合规

| 标准 | 支持情况 |
|------|---------|
| SOC 2 Type II | Anthropic 已认证 |
| GDPR | 数据处理协议 (DPA) |
| HIPAA | 需要 BAA |
| ISO 27001 | 审计中 |

## 部署架构建议

### 小团队（<20 人）

```
每人本地安装 Claude Code
共享 Git 中的 .claude/ 配置
统一 API Key 管理
```

### 中型团队（20-100 人）

```
统一配置模板 + 自动化分发
SSO 集成
成本监控仪表板
月度用量报告
```

### 大型组织（>100 人）

```
企业版部署
SSO + RBAC
全量审计日志
与现有 SIEM 系统集成
专属客户成功经理
```
