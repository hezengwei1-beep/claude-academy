# 第三章 Managed Agents 与远程调度

## 什么是 Managed Agents

Managed Agents（远程代理）让你**调度 Claude Code 在云端自动执行任务**，无需本地机器保持运行。

## /schedule 命令

### 创建定时任务

```
/schedule create --cron "0 9 * * 1-5" --prompt "检查昨天到现在的 CI 失败，分析根因，在 Slack #dev 频道发送报告"
```

每个工作日早上 9 点自动执行。

### 管理定时任务

```
/schedule list              # 列出所有定时任务
/schedule run <id>          # 手动触发一次
/schedule delete <id>       # 删除任务
```

### Cron 表达式速查

| 表达式 | 含义 |
|--------|------|
| `0 9 * * 1-5` | 工作日 9:00 |
| `0 18 * * 5` | 每周五 18:00 |
| `0 0 1 * *` | 每月 1 号 0:00 |
| `0 */4 * * *` | 每 4 小时 |
| `30 8 * * 1` | 每周一 8:30 |

## 实用调度 Recipes

### Recipe: 每日代码质量报告

```
/schedule create --cron "0 9 * * 1-5" --prompt "
生成今日代码质量报告：
1. 统计昨天的 commit 数量和改动行数
2. 检查是否有未解决的 lint 警告
3. 运行测试并报告覆盖率
4. 检查是否有过期的依赖
输出 Markdown 报告。
"
```

### Recipe: 每周安全扫描

```
/schedule create --cron "0 10 * * 1" --prompt "
执行每周安全扫描：
1. npm audit 检查依赖漏洞
2. 搜索代码中的硬编码密钥模式
3. 检查 .env.example 与实际环境变量是否匹配
4. 生成安全报告和修复建议
"
```

### Recipe: 每月依赖更新

```
/schedule create --cron "0 10 1 * *" --prompt "
检查所有过期的 npm 依赖：
1. 列出可更新的包（major/minor/patch 分开）
2. 对 major 更新，检查 changelog 中的 breaking changes
3. 建议安全的更新策略
4. 生成一个更新计划
"
```

## Headless 模式

### 完全无人值守执行

```bash
# 非交互模式，输出到文件
claude -p "运行所有测试并生成覆盖率报告" --output text > report.txt

# JSON 输出，方便程序解析
claude -p "分析 src/ 的代码复杂度" --output json > complexity.json
```

### 在 cron 中使用

```bash
# crontab -e
0 9 * * 1-5 cd ~/Projects/my-app && claude -p "生成每日站会报告" > /tmp/standup.md 2>&1
```

### 在脚本中使用

```bash
#!/bin/bash
# daily-check.sh

cd ~/Projects/my-app

# 检查测试
TEST_RESULT=$(claude -p "运行 npm test，只告诉我通过/失败和失败数量" --output text 2>/dev/null)

# 如果失败，发通知
if echo "$TEST_RESULT" | grep -q "失败"; then
  curl -X POST "$SLACK_WEBHOOK" -d "{\"text\": \"测试失败: $TEST_RESULT\"}"
fi
```

## Triggers（事件触发器）

### Webhook 触发

配置当外部事件发生时自动启动 Claude Code 任务：

```
/schedule create --trigger webhook --prompt "有新的 Issue 被创建，请分析并分类"
```

### 与 GitHub Actions 结合

```yaml
# 当 Issue 创建时触发 Claude 分析
on:
  issues:
    types: [opened]

jobs:
  analyze:
    runs-on: ubuntu-latest
    steps:
      - run: |
          claude -p "分析 Issue #${{ github.event.issue.number }}: ${{ github.event.issue.title }}"
```
