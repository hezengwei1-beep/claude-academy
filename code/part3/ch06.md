# 第六章 Hooks：事件驱动自动化

## Hooks 是什么

Hooks 是 Claude Code 的事件系统。当特定事件发生时，自动执行你定义的 shell 命令。

类比：就像 Git 的 pre-commit hook，但触发的事件是 Claude Code 的操作。

## 配置方式

在 `settings.json` 中添加 `hooks` 字段：

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "command": "npx eslint --fix $FILE_PATH",
        "description": "Auto-fix lint after file changes"
      }
    ],
    "PreToolUse": [
      {
        "matcher": "Bash",
        "command": "echo '即将执行命令: $COMMAND'",
        "description": "Log bash commands"
      }
    ]
  }
}
```

## 可用的 Hook 事件

| 事件 | 触发时机 |
|------|---------|
| `PreToolUse` | 工具调用之前 |
| `PostToolUse` | 工具调用之后 |
| `Notification` | 收到通知时 |
| `TeammateIdle` | Agent Teams 队友空闲时 |
| `TaskCreated` | 任务创建时 |
| `TaskCompleted` | 任务完成时 |

## 实战：自动格式化

每次 Claude 修改文件后自动运行 Prettier：

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "command": "npx prettier --write $FILE_PATH 2>/dev/null || true",
        "description": "Auto-format on file change"
      }
    ]
  }
}
```

## 实战：自动测试

每次文件修改后运行相关测试：

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "command": "npx vitest run --reporter=verbose --changed 2>&1 | head -30",
        "description": "Run related tests after changes"
      }
    ]
  }
}
```

## 实战：安全护栏

阻止危险命令：

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "command": "echo '$COMMAND' | grep -qE '(rm -rf|drop table|--force)' && echo 'BLOCKED: 危险命令' && exit 1 || exit 0",
        "description": "Block dangerous commands"
      }
    ]
  }
}
```

Hook 返回非零退出码时，对应的操作会被阻止。

## 实战：提交前检查

在用户请求提交之前自动检查：

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash(git commit)",
        "command": "npm run lint && npm test",
        "description": "Run lint and tests before commit"
      }
    ]
  }
}
```

## Hook 编写注意事项

### 1. 保持快速

Hook 是同步执行的。超过 30 秒的 Hook 会明显影响体验。

### 2. 幂等

Hook 可能被多次触发，确保多次执行和一次执行效果相同。

### 3. 静默失败

非关键的 Hook 应该静默失败（`|| true`），不要阻塞主流程：

```bash
npx prettier --write $FILE_PATH 2>/dev/null || true
```

### 4. 可用的环境变量

| 变量 | 说明 |
|------|------|
| `$FILE_PATH` | 被操作的文件路径 |
| `$COMMAND` | 被执行的命令 |
| `$TOOL_NAME` | 工具名称 |
| `$PROJECT_DIR` | 项目根目录 |
