# 第八章 CI/CD 集成与自动化

## 非交互模式在 CI 中的使用

Claude Code 的 `-p` 参数支持非交互模式，可以嵌入 CI/CD 流水线：

```bash
claude -p "审查 git diff main...HEAD 的改动，输出 JSON 格式的问题列表" --output json
```

## GitHub Actions 集成

### 自动代码审查

```yaml
# .github/workflows/claude-review.yml
name: Claude Code Review
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install Claude Code
        run: npm install -g @anthropic-ai/claude-code

      - name: Run Review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          DIFF=$(git diff origin/main...HEAD)
          claude -p "审查以下代码改动，只报告 Critical 和 High 级别的问题：\n$DIFF" \
            --output text > review.md

      - name: Comment on PR
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const review = fs.readFileSync('review.md', 'utf8');
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: `## Claude Code Review\n\n${review}`
            });
```

### 自动生成变更日志

```yaml
- name: Generate Changelog
  run: |
    claude -p "基于 git log --oneline main...HEAD 生成本次发布的变更日志，分为 Features/Fixes/Other" \
      > CHANGELOG-draft.md
```

## 本地 Git Hooks 集成

### pre-commit

```bash
#!/bin/bash
# .git/hooks/pre-commit

# 用 Claude 检查暂存的改动
STAGED=$(git diff --cached --name-only)
if [ -n "$STAGED" ]; then
  claude -p "快速检查以下文件的明显问题（不要输出没问题的文件）：$STAGED" 2>/dev/null
fi
```

### commit-msg

```bash
#!/bin/bash
# .git/hooks/commit-msg

MSG=$(cat "$1")
# 用 Claude 检查 commit message 质量
FEEDBACK=$(claude -p "这个 commit message 是否清晰描述了'为什么'而不是'是什么'？消息：$MSG。只回答 OK 或给出改进建议。" 2>/dev/null)
echo "$FEEDBACK"
```

## 定时任务

### /loop 命令

在 Claude Code 中设置定时循环：

```
/loop 5m npm test
```

每 5 分钟运行一次测试。适合：
- 监控构建状态
- 定期检查代码质量
- 等待异步操作完成

### /schedule 命令

创建远程调度任务：

```
/schedule create --cron "0 9 * * 1" --prompt "生成上周的代码质量报告"
```

每周一早上 9 点自动生成报告。

## 批量处理

### /batch

对多个独立文件并行处理：

```
/batch src/api/*.ts "给每个文件添加 JSDoc 注释"
```

Claude 会并行处理所有匹配的文件。

### 管道批量

```bash
find src -name "*.test.ts" | xargs -I {} claude -p "检查 {} 的测试覆盖率是否足够"
```

## 安全注意事项

### CI 中的 API Key

- 使用 GitHub Secrets 存储 `ANTHROPIC_API_KEY`
- 不要在日志中打印 API Key
- 设置 API Key 的使用限额

### 成本控制

CI 中每次 PR 触发审查会消耗 API tokens。建议：
- 只在 PR 创建时审查，不在每次 push 时
- 对小改动跳过审查（如 < 10 行）
- 设置月度预算上限

### 输出安全

Claude 的输出可能包含敏感信息。在公开的 PR 评论中：
- 不要输出文件的完整内容
- 不要输出环境变量
- 只输出问题描述和建议
