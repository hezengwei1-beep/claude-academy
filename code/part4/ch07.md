# 第七章 IDE 集成

## VS Code 扩展

### 安装

在 VS Code 扩展市场搜索 "Claude Code" 并安装。或在终端：

```bash
code --install-extension anthropic.claude-code
```

### 三种使用方式

#### 1. 侧边栏面板

点击 VS Code 左侧活动栏中的 Claude 图标，打开对话面板。

**优势**：不离开 IDE，边写代码边提问。

**典型用法**：
```
看一下我当前打开的文件有什么问题

帮我给选中的函数加 TypeScript 类型

这个报错是什么意思？怎么修？
```

#### 2. 内联操作

选中代码 → `Cmd+Shift+P` → 输入 "Claude" → 选择操作：

| 操作 | 快捷方式 | 作用 |
|------|---------|------|
| Ask Claude | 选中 → 右键 → Ask Claude | 对选中代码提问 |
| Explain | Cmd+Shift+P → Claude: Explain | 解释选中代码 |
| Fix | Cmd+Shift+P → Claude: Fix | 修复选中代码的问题 |
| Refactor | Cmd+Shift+P → Claude: Refactor | 重构选中代码 |

**实操示例**：

```
1. 选中一段有 bug 的代码
2. Cmd+Shift+P → Claude: Fix
3. Claude 分析问题并给出修复
4. 预览 diff → 接受或拒绝
```

#### 3. 内置终端

直接在 VS Code 终端中运行 `claude`，获得完整的 CLI 体验：

```bash
# VS Code 终端中
claude
```

这种方式功能最完整（100%），但需要切换到终端窗口。

### VS Code 配置优化

在 VS Code settings.json 中添加：

```json
{
  // Claude Code 扩展设置
  "claude-code.model": "claude-sonnet-4-6",
  "claude-code.autoSave": true,

  // 推荐的编辑器设置（配合 Claude Code）
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": "explicit"
  }
}
```

### VS Code 常用工作流

**工作流 1：边写边审**
```
1. 写一段代码
2. 选中 → Ask Claude："这段代码有安全问题吗？"
3. 根据建议修改
4. 继续写下一段
```

**工作流 2：快速修 Bug**
```
1. 看到红色波浪线（TypeScript 报错）
2. 选中出错行 → Claude: Fix
3. 预览修复 → 接受
```

**工作流 3：理解遗留代码**
```
1. 打开一个不熟悉的文件
2. 选中整个函数 → Claude: Explain
3. Claude 解释每一步逻辑
```

## JetBrains 扩展

### 安装

在 JetBrains 插件市场搜索 "Claude Code"。支持全系列 IDE：

| IDE | 支持 |
|-----|------|
| IntelliJ IDEA | ✅ |
| WebStorm | ✅ |
| PyCharm | ✅ |
| GoLand | ✅ |
| Rider | ✅ |
| CLion | ✅ |

### 使用方式

与 VS Code 类似：
- **Tool Window**：右侧面板打开 Claude 对话
- **上下文菜单**：选中代码 → 右键 → Claude 操作
- **内置终端**：运行 `claude` 获得完整 CLI

### JetBrains 特有优势

- **更好的代码导航集成**：Claude 可以利用 JetBrains 的索引
- **重构支持**：Claude 的重构建议可以通过 JetBrains 的重构工具执行
- **数据库工具集成**：配合 DataGrip 或数据库面板

## 终端 vs IDE 扩展选择

| 维度 | 终端 Claude Code | IDE 扩展 |
|------|-----------------|---------|
| 功能完整度 | 100% | 80-90% |
| Agent Teams | ✅ 完整支持 | ⚠️ 部分支持 |
| MCP Servers | ✅ 完整支持 | ⚠️ 部分支持 |
| Hooks | ✅ 完整支持 | ❌ 不支持 |
| 文件操作 | 全功能 | 受 IDE 沙箱限制 |
| 多文件改动 | 方便 | 方便 |
| 代码补全 | 基于对话 | 基于上下文（更快） |
| 适合场景 | 大型任务、多代理、CI/CD | 快速的内联操作、日常编码 |

**最佳实践**：两者配合使用：
- 日常编码中的快速提问 → IDE 扩展
- 多文件重构、代码审查、Agent Teams → 终端

## Claude Code Web

通过浏览器使用 Claude Code，无需本地安装：

- **访问方式**：claude.ai/code
- **运行环境**：云端 VM
- **适用场景**：
  - 没有本地开发环境（如平板电脑）
  - 临时使用（不想装软件）
  - 演示和教学
- **限制**：
  - 文件持久化依赖云端存储
  - 网络延迟比本地略高
  - 部分本地工具不可用

## 团队协作配置

### 共享 vs 个人

| 文件 | Git 提交？ | 作用 |
|------|-----------|------|
| `.claude/settings.json` | ✅ 是 | 团队统一配置（模型、权限） |
| `.claude/commands/*.md` | ✅ 是 | 团队统一 Slash Commands |
| `.mcp.json` | ✅ 是 | 团队共享的 MCP Servers |
| `CLAUDE.md` | ✅ 是 | 项目约定和规则 |
| `.claude/settings.local.json` | ❌ 否 | 个人偏好覆盖 |
| `.claude/todos.json` | ❌ 否 | 个人任务 |

### .gitignore 建议

```gitignore
# Claude Code - 个人文件
.claude/settings.local.json
.claude/todos.json
.claude/projects/

# Claude Code - 缓存
.claude/cache/
```

### 团队 Onboarding 清单

新成员加入后：

```
1. npm install -g @anthropic-ai/claude-code
2. 配置 API Key（团队统一发放或个人申请）
3. git clone 项目 → .claude/ 和 CLAUDE.md 自动生效
4. 运行 claude → 验证 Slash Commands 可用（/review、/commit 等）
5. 如有 MCP Servers → 检查 /mcp 状态
6. 个人偏好写到 .claude/settings.local.json
```

### 团队统一模型

```json
// .claude/settings.json（提交到 Git）
{
  "model": "claude-sonnet-4-6",
  "permissions": {
    "allow": ["Read", "Glob", "Grep", "Bash(npm test)", "Bash(npm run lint)"],
    "deny": ["Bash(git push --force *)", "Bash(rm -rf *)"]
  }
}
```

这样团队每个人的 Claude Code 行为一致：同模型、同权限、同安全策略。
