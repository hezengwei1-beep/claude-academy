# 第七章 IDE 集成

## VS Code 扩展

### 安装

在 VS Code 扩展市场搜索 "Claude Code" 并安装。

### 使用方式

安装后，你可以：

1. **侧边栏面板**：在 VS Code 侧边栏打开 Claude Code 面板，直接对话
2. **内联操作**：选中代码 → 右键 → "Ask Claude" → 输入指令
3. **终端集成**：在 VS Code 内置终端中直接运行 `claude`

### 常用场景

**选中代码请求解释**：
1. 选中一段代码
2. 右键 → Ask Claude
3. 输入 "解释这段代码"

**选中代码请求修改**：
1. 选中一段代码
2. 右键 → Ask Claude
3. 输入 "重构这段代码，提取重复逻辑"

## JetBrains 扩展

### 安装

在 JetBrains 插件市场搜索 "Claude Code" 并安装。支持 IntelliJ IDEA、WebStorm、PyCharm 等。

### 使用方式

与 VS Code 类似：侧边栏面板 + 内联操作。

## 终端 vs IDE 扩展

| 维度 | 终端 Claude Code | IDE 扩展 |
|------|-----------------|---------|
| 功能完整度 | 100% | 80-90% |
| Agent Teams | 支持 | 部分支持 |
| MCP | 完整支持 | 部分支持 |
| 文件操作 | 全功能 | 受 IDE 限制 |
| 适合场景 | 大型任务、多代理 | 快速的内联操作 |

**建议**：日常小修改用 IDE 扩展，大型任务用终端。两者不冲突，可以同时使用。

## Claude Code Web

Claude Code 也有 Web 版本，通过浏览器使用：

- 无需本地安装
- 在云 VM 中运行
- 适合没有本地开发环境的场景
- 功能与 CLI 基本一致

## 团队协作配置

### 共享项目配置

把以下文件提交到 Git，团队成员 clone 后自动生效：

```
.claude/settings.json    → 项目配置
.mcp.json               → MCP Server 配置
CLAUDE.md               → 项目指令
```

### .gitignore 建议

```
# Claude Code
.claude/settings.local.json   # 个人配置
.claude/todos.json            # 个人任务
```

### 团队统一模型

在项目 settings.json 中指定模型，确保团队一致性：

```json
{
  "model": "claude-sonnet-4-6"
}
```
