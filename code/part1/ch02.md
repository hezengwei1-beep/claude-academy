# 第二章 核心命令与快捷键

## 斜杠命令全览

Claude Code 内置 15+ 个 `/` 命令。掌握它们是高效使用的基础。

### 对话管理（最常用）

| 命令 | 作用 | 使用时机 |
|------|------|---------|
| `/clear` | 清除对话历史，从零开始 | 切换到全新任务时 |
| `/compact` | 智能压缩对话（保留关键信息） | 对话 >20 轮后、Claude 变慢时 |
| `/resume` | 恢复之前保存的会话 | 继续昨天的工作 |
| `/rewind` | 回滚到上一个检查点 | Claude 的改动搞坏了什么 |

#### /compact —— 最被低估的命令

```
/compact
```

对话 20 轮后，上下文中累积了大量的文件内容、命令输出、中间讨论。`/compact` 让 Claude 做一次"总结"：保留关键信息，丢弃已完成的中间步骤。

**带提示的压缩**：
```
/compact 保留关于数据库迁移的所有讨论，其他可以压缩
```

#### /resume —— 续接上次会话

```bash
claude --resume
```

Claude Code 自动保存会话。/resume 列出最近的会话供你选择恢复。

**最佳实践**：每个功能/bug 用一个会话。完成后开新会话。

#### /rewind —— 时光倒流

回滚到上一个检查点。适用于 Claude 的修改引入了新问题时。

### 模式切换

| 命令 | 作用 | 说明 |
|------|------|------|
| `/plan` | 规划模式 | Claude 只分析不执行，先出方案 |
| `/fast` | 快速模式 | 同模型但更快输出 |

#### /plan —— 先想后做

```
/plan
重构 src/auth/ 模块
```

Plan 模式下 Claude 只分析、不执行。确认方案后再动手。适合大型重构。

### 工具和扩展

| 命令 | 作用 |
|------|------|
| `/mcp` | 管理 MCP Servers |
| `/agents` | 管理子代理 |
| `/loop` | 定时循环任务 |

### 其他

| 命令 | 作用 |
|------|------|
| `/help` | 显示帮助 |
| `/exit` | 退出 |

## 键盘快捷键

### 基础

| 快捷键 | 作用 |
|--------|------|
| `Enter` | 发送 |
| `Shift+Enter` | 换行（不发送） |
| `Ctrl+C` | 中断当前操作 |
| `Ctrl+D` | 退出 |
| `Esc` | 取消输入 |
| `Up/Down` | 浏览历史 |
| `Tab` | 文件路径补全 |

### Agent Teams 专用

| 快捷键 | 作用 |
|--------|------|
| `Shift+Down/Up` | 切换 teammate |
| `Ctrl+T` | 任务列表 |
| `Shift+Tab` | Delegate Mode |

## CLI 参数

```bash
claude                              # 交互模式
claude -p "任务"                    # 非交互模式
claude -p "任务" --output json      # JSON 输出
claude --model opus                 # 指定模型
claude --worktree feature-x         # Worktree 模式
claude --resume                     # 恢复会话
claude --version                    # 版本
claude --usage                      # 用量
```

## ! 前缀运行 Shell

```
! ls -la src/
! git log --oneline -5
! npm test
```

命令输出直接进入对话上下文，Claude 能看到并基于它回答。
