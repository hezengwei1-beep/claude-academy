# 第一章 安装与首次运行

## 安装

### macOS / Linux

```bash
npm install -g @anthropic-ai/claude-code
```

### 验证

```bash
claude --version
```

### 更新

```bash
npm update -g @anthropic-ai/claude-code
```

## 首次运行

```bash
cd ~/Projects/my-project
claude
```

首次启动会：
1. 引导你登录 Anthropic 账号（或配置 API Key）
2. 显示欢迎界面
3. 进入交互式对话

## 核心概念

### Claude Code 是什么

Claude Code 是一个**终端中的 AI 编码代理**。它不只是回答问题——它能直接：

- 读取你的文件
- 修改你的代码
- 运行 shell 命令
- 搜索代码库
- 执行 Git 操作
- 运行测试

### 它不是什么

- 不是 IDE 插件（虽然有 VS Code/JetBrains 扩展）
- 不是 ChatGPT 的替代品（它专注于编码）
- 不是自动驾驶（你是驾驶员，它是副驾驶）

## 第一个任务

在你的项目目录中启动 Claude Code，然后试试：

```
这个项目是做什么的？给我一个简要概述。
```

Claude 会自动读取 README、package.json、主要源文件等，然后给你一个项目概述。

### 再试几个

```
找出这个项目中所有的 TODO 注释

最近一次 git commit 改了什么？

src/auth/ 目录下的代码做了什么？
```

## 工作模式

Claude Code 有两种工作模式：

### 交互模式（默认）

```bash
claude
```

启动后进入对话界面，你一条一条地输入指令。

### 非交互模式

```bash
claude -p "列出 src/ 下所有 .ts 文件的行数统计"
```

`-p` 参数直接执行一个任务然后退出，适合脚本和自动化。

### Pipe 模式

```bash
git diff | claude -p "审查这些改动"
cat error.log | claude -p "分析这个错误日志"
```

把其他命令的输出通过管道传给 Claude。

## 权限模型

Claude Code 在执行操作前会请求你的许可：

| 操作类型 | 默认行为 |
|----------|---------|
| 读取文件 | 自动允许 |
| 搜索代码 | 自动允许 |
| 修改文件 | **需要确认** |
| 运行命令 | **需要确认** |
| Git 操作 | **需要确认** |

你可以选择：
- **Allow once**：本次允许
- **Allow always**：永久允许这类操作
- **Deny**：拒绝

后面的章节会详细讲如何配置权限策略。

## 退出

- 输入 `/exit` 或按 `Ctrl+C` 退出
- 输入 `/clear` 清除当前对话
- 输入 `/help` 查看所有命令
