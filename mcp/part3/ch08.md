
# 第八章 用 Python 从零构建 MCP Server

## 前置条件

- Python >= 3.10
- uv（推荐）或 pip

```bash
mkdir todo-mcp-server && cd todo-mcp-server
```

## FastMCP：最快的入门方式

Python SDK 提供了 `FastMCP` 类，用装饰器定义工具、资源和 prompts。

创建 `server.py`：

```python
from mcp.server.fastmcp import FastMCP
import json
from pathlib import Path

# 创建 Server 实例
mcp = FastMCP("todo-server")

# 持久化存储
TODO_FILE = Path.home() / ".mcp-todos.json"

def load_todos() -> list[dict]:
    if TODO_FILE.exists():
        return json.loads(TODO_FILE.read_text())
    return []

def save_todos(todos: list[dict]):
    TODO_FILE.write_text(json.dumps(todos, ensure_ascii=False, indent=2))


# ============ Tools ============

@mcp.tool()
def add_todo(title: str, priority: str = "medium") -> str:
    """添加一个新的待办事项。

    Args:
        title: 待办事项标题
        priority: 优先级，low/medium/high
    """
    todos = load_todos()
    todo = {
        "id": len(todos) + 1,
        "title": title,
        "priority": priority,
        "done": False,
    }
    todos.append(todo)
    save_todos(todos)
    return f"已添加待办事项 #{todo['id']}: {title} [{priority}]"


@mcp.tool()
def list_todos(show_done: bool = False) -> str:
    """列出所有待办事项。

    Args:
        show_done: 是否显示已完成的事项
    """
    todos = load_todos()
    if not show_done:
        todos = [t for t in todos if not t["done"]]

    if not todos:
        return "没有待办事项。"

    lines = []
    for t in todos:
        status = "✓" if t["done"] else "○"
        lines.append(f"{status} #{t['id']} [{t['priority']}] {t['title']}")
    return "\n".join(lines)


@mcp.tool()
def complete_todo(todo_id: int) -> str:
    """将一个待办事项标记为已完成。

    Args:
        todo_id: 待办事项的 ID
    """
    todos = load_todos()
    for t in todos:
        if t["id"] == todo_id:
            t["done"] = True
            save_todos(todos)
            return f"已完成: #{todo_id} {t['title']}"
    return f"未找到 ID 为 {todo_id} 的待办事项"


@mcp.tool()
def delete_todo(todo_id: int) -> str:
    """删除一个待办事项。

    Args:
        todo_id: 待办事项的 ID
    """
    todos = load_todos()
    todos = [t for t in todos if t["id"] != todo_id]
    save_todos(todos)
    return f"已删除待办事项 #{todo_id}"


# ============ Resources ============

@mcp.resource("todos://summary")
def todo_summary() -> str:
    """获取待办事项的统计摘要。"""
    todos = load_todos()
    total = len(todos)
    done = sum(1 for t in todos if t["done"])
    high = sum(1 for t in todos if t["priority"] == "high" and not t["done"])

    return json.dumps({
        "total": total,
        "done": done,
        "pending": total - done,
        "high_priority_pending": high,
    }, ensure_ascii=False, indent=2)


# ============ Prompts ============

@mcp.prompt()
def daily_review() -> str:
    """生成每日回顾提示。"""
    return """请帮我做每日待办事项回顾：
1. 先用 list_todos 查看所有未完成的事项
2. 按优先级排序，列出今天应该重点完成的
3. 对于拖延超过预期的事项，给出建议
4. 最后用 todo_summary 资源给出整体统计"""


# 启动
if __name__ == "__main__":
    mcp.run(transport="stdio")
```

## FastMCP 的魔法

注意代码中**没有手动定义 JSON Schema**。FastMCP 自动从以下信息生成：

1. **函数名** → 工具名称
2. **类型注解** → 参数类型
3. **Docstring** → 工具描述和参数描述
4. **默认值** → 可选参数

```python
@mcp.tool()
def add_todo(title: str, priority: str = "medium") -> str:
    """添加一个新的待办事项。

    Args:
        title: 待办事项标题
        priority: 优先级，low/medium/high
    """
```

自动生成的 Schema：
```json
{
  "name": "add_todo",
  "description": "添加一个新的待办事项。",
  "inputSchema": {
    "type": "object",
    "properties": {
      "title": { "type": "string", "description": "待办事项标题" },
      "priority": { "type": "string", "default": "medium", "description": "优先级，low/medium/high" }
    },
    "required": ["title"]
  }
}
```

## 进阶：异步工具

对于 I/O 密集型操作（API 调用、数据库查询），使用 `async`：

```python
import httpx

@mcp.tool()
async def fetch_url(url: str) -> str:
    """获取 URL 的内容。

    Args:
        url: 要获取的 URL
    """
    async with httpx.AsyncClient() as client:
        response = await client.get(url)
        return response.text[:5000]  # 限制返回长度
```

## 进阶：类型安全的参数

使用 Pydantic 模型定义复杂参数：

```python
from pydantic import BaseModel, Field

class TodoFilter(BaseModel):
    priority: str | None = Field(None, description="按优先级筛选")
    show_done: bool = Field(False, description="是否显示已完成")
    limit: int = Field(10, description="最多返回条数")

@mcp.tool()
def search_todos(filter: TodoFilter) -> str:
    """按条件搜索待办事项。"""
    todos = load_todos()
    if filter.priority:
        todos = [t for t in todos if t["priority"] == filter.priority]
    if not filter.show_done:
        todos = [t for t in todos if not t["done"]]
    return json.dumps(todos[:filter.limit], ensure_ascii=False)
```

## Python vs TypeScript 选择

| 维度 | Python | TypeScript |
|------|--------|-----------|
| SDK 成熟度 | Tier 1 | Tier 1 |
| 入门难度 | 低（FastMCP 装饰器） | 中 |
| 异步支持 | async/await | async/await |
| 类型安全 | 通过 Pydantic | 通过 Zod |
| 生态 | 数据科学/ML 生态丰富 | 前端/全栈生态丰富 |
| 分发 | uvx / pip | npx / npm |
| 性能 | 足够 | 略快 |

**经验法则**：如果你更熟悉哪个就用哪个。两者都是 Tier 1 SDK，能力完全相同。
