
# Workshop 1：从零用 Agent Teams 审查一个开源 PR

**预计耗时**：30-45 分钟
**前置条件**：Claude Code 已安装，Agent Teams 已启用
**目标**：亲手体验 3 人 Agent Teams 的完整工作流

## 准备工作（5 分钟）

### 1. 确认环境

```bash
# 检查版本
claude --version
# 应 >= 2.1.32

# 检查 Agent Teams 是否启用
echo $CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS
# 应输出 1
```

### 2. 选择一个开源项目

选一个你熟悉的开源项目，找一个中等大小的 PR（50-200 行改动）。如果没有，以下是一些建议：

- 你自己参与的任何 GitHub 项目
- 任何你感兴趣的开源项目的 recent merged PR

### 3. 克隆仓库

```bash
git clone <repo-url>
cd <repo-name>
```

## 第一阶段：单 Agent 审查（10 分钟）

先用单个 Claude Code 审查，作为对照基线。

```bash
claude
```

在 Claude Code 中输入：

```
请审查最近的 PR 改动（git diff main...HEAD 或查看最近的 commit）。
从安全性、性能、测试覆盖率三个角度分析。
记录你的发现和耗时。
```

**记录**：
- 发现了多少个问题？
- 哪些维度覆盖得深，哪些浅？
- 总耗时多少？

退出会话。

## 第二阶段：Agent Teams 审查（15 分钟）

现在用 3 人 Agent Teams 做同样的事。

```bash
claude
```

### Step 1：进入 Delegate Mode

```
进入 Delegate Mode。从现在开始你只做协调，不要自己读代码。
```

### Step 2：创建 3 人团队

```
创建一个 3 人代码审查团队来审查最近的改动：

1. security-reviewer
   - 职责：安全性审查
   - 重点：输入验证、认证授权、敏感数据处理、OWASP Top 10
   - 输出格式：[Critical/High/Medium/Low] 文件:行号 - 问题描述

2. perf-reviewer
   - 职责：性能审查
   - 重点：查询效率、内存使用、不必要的计算、缓存机会
   - 输出格式：同上

3. test-reviewer
   - 职责：测试覆盖率审查
   - 重点：缺失的测试用例、边界情况、mock 是否合理
   - 输出格式：列出缺失的测试场景

规则：
- 每人独立审查，发现与其他人职责相关的问题直接发消息给对方
- 完成后在任务列表标记完成，并给我发消息汇总
```

### Step 3：监控进度

用 `Shift+Down/Up` 切换查看每个 teammate 的工作。

用 `Ctrl+T` 查看任务列表进度。

观察：
- 三个 teammate 是否在同时工作？
- 有没有 teammate 之间的通信发生？

### Step 4：收集结果

三人都完成后：

```
请综合三位审查者的发现，生成最终审查报告：
1. 按严重度排序
2. 标注交叉发现（多个审查者提到的关联问题）
3. 与我之前单 agent 审查的结果对比
```

## 第三阶段：对比分析（5 分钟）

## 复盘检查清单

- [ ] Agent Teams 是否确实并行工作了？
- [ ] 是否观察到了 teammate 之间的通信？
- [ ] Delegate Mode 下 Lead 是否只做了协调？
- [ ] 综合报告是否比单 agent 报告更全面？
- [ ] 总耗时是否明显少于单 agent？
- [ ] 你觉得额外的 token 成本值得吗？

## 常见问题

**Q：teammate 没有开始工作**
A：检查 Agent Teams 是否正确启用，尝试重新 spawn。

**Q：三个 teammate 都在读同一个文件**
A：正常，审查场景下读取是安全的（只要不同时修改）。

**Q：没有观察到 teammate 之间的通信**
A：可能 PR 改动较小，没有产生交叉发现。尝试一个更大的 PR。

## 下一步

完成这个 Workshop 后，你已经掌握了 Agent Teams 的基本用法。建议的进阶练习：

1. 尝试用 4-5 人团队审查更大的 PR
2. 尝试在审查中加入"质疑者"角色——专门挑战其他人的结论
3. 尝试用 Plan → Team 工作流做一次功能开发
