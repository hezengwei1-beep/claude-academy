
# Workshop 3：用 Agent Teams 并行开发一个功能

**预计耗时**：45-60 分钟
**前置条件**：Agent Teams 启用，有一个 Node.js/TypeScript 项目
**目标**：用 Delegate Mode + Plan → Team 工作流完成一个跨层功能

## 任务定义

我们要为一个 Web 应用添加"用户偏好设置"功能：
- 数据库层：新增 user_preferences 表
- API 层：CRUD endpoints
- 测试层：单元测试和集成测试

这是一个天然可以并行化的三层任务。

## 第一阶段：Plan（10 分钟）

```bash
cd your-project
claude
```

```
/plan

我需要添加一个"用户偏好设置"功能。需要：
1. 数据库层：user_preferences 表（user_id, key, value, created_at, updated_at）
2. API 层：GET/PUT/DELETE /api/users/:id/preferences
3. 测试层：单元测试 + 集成测试

请分析当前代码库的结构，制定执行计划，包括：
- 需要新建/修改的文件清单
- 文件之间的依赖顺序
- 建议的 3 人团队分工
- 每人的文件边界
```

审查 Claude 的计划，确认后继续。

## 第二阶段：Team Setup（5 分钟）

```
进入 Delegate Mode。

按计划创建 3 人开发团队：

1. db-dev
   - 负责：数据库迁移文件、model 定义
   - 文件范围：[根据你的项目结构填写]
   - 完成标准：迁移可运行，model 导出可用

2. api-dev
   - 负责：API 路由、controller、service
   - 文件范围：[根据你的项目结构填写]
   - 依赖：db-dev 完成 model 后开始
   - 完成标准：CRUD endpoints 可调用

3. test-dev
   - 负责：测试文件
   - 文件范围：[根据你的项目结构填写]
   - 可以先写 model 层的单元测试（与 db-dev 并行）
   - API 测试等 api-dev 完成后再写
   - 完成标准：所有测试通过

任务依赖：
- db-dev 无依赖，立即开始
- api-dev 依赖 db-dev
- test-dev 部分可并行，部分依赖 api-dev

规则：
- 每人只改自己范围的文件
- 修改接口定义前先发消息给依赖方
- 完成后标记任务，发消息给我
```

## 第三阶段：监控执行（20-30 分钟）

### 你要做的事

1. **每 5 分钟** 用 `Ctrl+T` 查看任务进度
2. **关注通信**：db-dev 完成 model 时是否通知了 api-dev？
3. **处理问题**：如果有 teammate 卡住了，介入给方向

### 关键时间点

- **~5 分钟**：db-dev 应该完成迁移文件和 model
- **~10 分钟**：api-dev 开始工作，test-dev 已经在写 model 测试
- **~20 分钟**：api-dev 完成大部分 endpoints
- **~30 分钟**：test-dev 完成所有测试

### 如果出现问题

**db-dev 和 api-dev 的接口不匹配**：
```
给 api-dev 和 db-dev 发消息：
请确认 model 的接口定义。db-dev 先发当前的类型定义，api-dev 确认后再继续。
```

**test-dev 的测试一直失败**：
```
给 test-dev 发消息：暂停，让我看一下失败原因。
```
然后 `Shift+Down` 切到 test-dev 查看问题。

## 第四阶段：验证和综合（10 分钟）

所有 teammate 完成后：

```
请做最终验证：
1. 运行所有测试，确认全部通过
2. 检查三人的代码是否接口一致
3. 检查是否有遗漏的文件（如 index.ts 的导出）
4. 生成一份变更总结
```

## 复盘检查清单

- [ ] Plan 阶段的计划是否合理？执行时有多少偏差？
- [ ] 文件边界是否有效？有没有出现冲突？
- [ ] 任务依赖是否正确触发？db-dev 完成后 api-dev 是否自动开始？
- [ ] teammate 之间是否产生了有价值的通信？
- [ ] Delegate Mode 下你是否成功避免了自己动手？
- [ ] 最终测试是否全部通过？
- [ ] 对比估算：这个功能如果用单 agent 需要多久？
