
# 前言

## 为什么写这本书

2026 年 4 月，我坐在终端前，看着 5 个 Claude Code 实例同时工作——一个在重构认证模块，一个在写测试，一个在审查安全性，两个在互相讨论一个边界情况的处理方式。20 分钟后，一个原本需要半天的任务完成了。

那一刻我意识到：AI 协作的范式已经从"一个聪明的助手"变成了"一支可以编排的团队"。

但现实是，绝大多数人还在用最基础的方式使用 Claude——一问一答的聊天模式。原因很简单：**没有一个地方把 Claude 的协作能力从入门到精通讲清楚**。

官方文档准确但分散。社区经验深刻但零碎。博客文章有价值但各说各话。

这本书试图解决这个问题。

## 这本书是什么

这是一本**从零到一，再从一到一百**的完整指南。

- **从零到一**：如果你从未用过 Claude Cowork 或 Claude Code，Part 1-2 会手把手带你上手
- **从一到十**：如果你已经在用单个 Claude Code，Part 3-4 会教你组建多代理团队
- **从十到一百**：如果你已经用过 Agent Teams，Part 5-6 的案例、成本优化和反模式会帮你避坑

## 这本书不是什么

- **不是 Claude Code 的完整手册** —— 单 agent 的基础使用不在本书重点范围内
- **不是 API 编程教程** —— 虽然有 Agent SDK 章节，但本书主要面向 CLI 和 Desktop 用户
- **不是一成不变的** —— Claude 的功能在快速迭代中，本书基于 2026 年 4 月的产品状态

## 谁应该读这本书

| 你是 | 推荐阅读路径 |
|------|-------------|
| 非技术用户（产品、运营、管理） | Part 1 → Part 2 → 附录 A |
| 开发者新手 | Part 1 → Part 3 (Ch9-11) → Part 4 (Ch15-16) |
| 进阶开发者 | Part 3 全部 → Part 5 → Part 6 |
| 技术管理者 | Ch1-3 → Part 5 (案例) → Ch22 (成本) |

## 本书的信息来源

本书内容基于三个来源：

1. **Anthropic 官方文档和博客** —— 产品能力、架构设计、官方案例
2. **社区专家的实践经验** —— 来自 Medium、GitHub、Dev.to 等平台的深度文章
3. **真实案例数据** —— Anthropic 内部实践、企业部署报告、社区实验

每个案例中的数据和结论都标注了来源，你可以在附录 C 中找到完整的链接索引。

## 如何阅读这本书

**每一章都标注了难度级别**：小白 / 进阶 / 专家 / 所有。你可以：

- **顺序阅读**：从头到尾，循序渐进
- **跳读**：根据你的水平直接跳到对应级别的章节
- **查阅**：把附录当作速查手册，需要时翻阅

每章末尾有**小结**（核心要点回顾）和**练习**（动手验证），帮助你巩固知识。

## 一个提醒

Agent Teams 截至本书写作时仍是**实验性功能**。这意味着：

- 功能可能在后续版本中变化
- 某些操作可能不如预期稳定
- 本书描述的某些细节可能在你阅读时已经更新

但核心理念——**多代理协作的架构思想、团队编排的最佳实践、成本与质量的权衡**——这些不会因为功能迭代而过时。

让我们开始。
