## Claude Squad 详解

Claude Squad 是社区最大的开源多代理管理工具，值得深入了解。

### 核心概念

Claude Squad 在终端中用 tmux 管理多个 Claude Code（或其他 AI agent）实例。每个实例运行在独立的 Git worktree 中。

### 安装

```bash
# macOS
brew install smtg-ai/tap/claude-squad

# 或从源码
go install github.com/smtg-ai/claude-squad@latest
```

### 使用

```bash
# 启动 Claude Squad
cs

# 创建新实例
cs new --name "feature-auth" --prompt "实现 JWT 认证"

# 查看所有实例
cs list

# 切换实例
cs switch feature-auth
```

### 支持的 AI 工具

Claude Squad 不只支持 Claude Code：
- Claude Code
- OpenAI Codex
- Aider
- OpenCode
- Amp
- 任何终端 AI agent

### 与 Agent Teams 的对比

| 维度 | Agent Teams | Claude Squad |
|------|-------------|-------------|
| 代理间通信 | ✅ Mailbox | ❌ |
| 共享任务列表 | ✅ | ❌ |
| 界面 | 终端内嵌 | tmux 窗口 |
| Diff 预览 | ❌ | ✅ |
| 多 AI 支持 | ❌ Claude only | ✅ 多种 |
| Git 隔离 | 可选 worktree | 默认 worktree |
| 学习成本 | 中 | 中 |

**选择建议**：
- 需要代理间通信和协调 → Agent Teams
- 需要管理多种 AI 工具的并行实例 → Claude Squad
- 需要可视化 diff 预览 → Claude Squad

## oh-my-claudecode

一个社区开发的 "swarm orchestration" 框架，用于编排大规模的多代理任务。

### 核心理念

通过 Skill 文件定义编排策略，一个 slash command 启动多个 agents：

```yaml
# swarm-review.skill
agents:
  - name: security
    model: opus
    focus: "OWASP Top 10"
  - name: performance
    model: sonnet
    focus: "query optimization"
  - name: tests
    model: haiku
    focus: "coverage gaps"
```

### 适用场景

- 需要重复运行的标准化多代理工作流
- 团队共享的审查/测试编排配置
- 需要跨项目复用的 agent 配置

### 风险提示

swarm 编排如果配置不当，很容易导致本书 Ch21 中描述的"49 agents 并行灾难"。**务必设置 token budget 和超时限制**。
