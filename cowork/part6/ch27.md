
# 第二十七章 Agent SDK：编程式构建多代理系统

前面的章节都是通过 CLI 和自然语言来使用多代理功能。本章介绍另一条路径——用代码构建自定义的多代理系统。

## Agent SDK 是什么

Claude Code Agent SDK 是一个编程接口，让你用 TypeScript/JavaScript 代码来定义和编排 agents。

**CLI 方式** vs **SDK 方式**：

| 维度 | CLI（自然语言） | SDK（代码） |
|------|----------------|-------------|
| 交互方式 | 终端输入自然语言 | 编写 TypeScript 代码 |
| 灵活度 | 高（自然语言什么都能说） | 更高（可编程控制） |
| 可重复性 | 低（每次手动输入） | 高（代码即配置） |
| 集成能力 | 有限 | 可嵌入任何 Node.js 应用 |
| 适用场景 | 日常开发、临时任务 | 产品集成、自动化流水线 |

## 安装

```bash
npm install @anthropic-ai/claude-code
```

## 基础用法：单 Agent

```typescript
import { Agent } from '@anthropic-ai/claude-code';

// 创建一个 agent
const agent = new Agent({
  model: 'claude-sonnet-4-6',
  systemPrompt: '你是一个代码审查专家，专注安全性审查。',
  tools: ['Read', 'Grep', 'Bash'],
});

// 运行任务
const result = await agent.run({
  prompt: '审查 src/auth/ 目录下的所有文件，报告安全问题。',
  workingDirectory: '/path/to/project',
});

console.log(result.output);
```

## 多 Agent 编排：Subagents

```typescript
import { Agent, SubAgent } from '@anthropic-ai/claude-code';

// 主 agent
const lead = new Agent({
  model: 'claude-opus-4-6',
  systemPrompt: '你是团队负责人，协调其他 agents 的工作。',
});

// 定义 subagents
const securityReviewer = new SubAgent({
  name: 'security-reviewer',
  model: 'claude-sonnet-4-6',
  systemPrompt: '你专注安全性审查。只报告 OWASP Top 10 相关的问题。',
  tools: ['Read', 'Grep'],
});

const perfReviewer = new SubAgent({
  name: 'perf-reviewer',
  model: 'claude-sonnet-4-6',
  systemPrompt: '你专注性能审查。关注 N+1 查询、内存泄漏、不必要的计算。',
  tools: ['Read', 'Grep', 'Bash'],
});

// 并行执行
const [securityResult, perfResult] = await Promise.all([
  securityReviewer.run({ prompt: '审查 src/auth/ 的安全性' }),
  perfReviewer.run({ prompt: '分析 src/auth/ 的性能' }),
]);

// 主 agent 综合
const summary = await lead.run({
  prompt: `综合以下审查结果：
    安全审查：${securityResult.output}
    性能审查：${perfResult.output}
    生成最终报告。`,
});
```

## 高级模式：自定义工具

```typescript
import { Agent, Tool } from '@anthropic-ai/claude-code';

// 定义自定义工具
const databaseQuery: Tool = {
  name: 'query_database',
  description: '执行只读 SQL 查询',
  parameters: {
    type: 'object',
    properties: {
      sql: { type: 'string', description: 'SQL 查询语句' },
    },
    required: ['sql'],
  },
  execute: async ({ sql }) => {
    // 你的数据库查询逻辑
    // 注意：只允许 SELECT，禁止 DROP/DELETE/UPDATE
    if (!sql.trim().toUpperCase().startsWith('SELECT')) {
      throw new Error('Only SELECT queries are allowed');
    }
    const result = await db.query(sql);
    return JSON.stringify(result);
  },
};

const analyst = new Agent({
  model: 'claude-sonnet-4-6',
  tools: ['Read', databaseQuery],
  systemPrompt: '你是数据分析师，可以查询数据库来回答问题。',
});
```

## 实战：自动化 PR 审查流水线

以下是一个可以集成到 CI/CD 的完整示例：

```typescript
import { Agent, SubAgent } from '@anthropic-ai/claude-code';

interface ReviewResult {
  reviewer: string;
  issues: Array<{
    severity: 'Critical' | 'High' | 'Medium' | 'Low';
    file: string;
    line: number;
    description: string;
  }>;
}

async function reviewPR(prDiff: string): Promise<ReviewResult[]> {
  // 定义三个专业审查 agent
  const reviewers = [
    {
      name: 'security',
      prompt: '从安全角度审查以下 diff',
      systemPrompt: '你是安全审查专家。输出 JSON 格式的问题列表。',
    },
    {
      name: 'performance',
      prompt: '从性能角度审查以下 diff',
      systemPrompt: '你是性能审查专家。输出 JSON 格式的问题列表。',
    },
    {
      name: 'testing',
      prompt: '从测试覆盖角度审查以下 diff',
      systemPrompt: '你是测试审查专家。输出 JSON 格式的问题列表。',
    },
  ];

  // 并行执行
  const results = await Promise.all(
    reviewers.map(async (r) => {
      const agent = new SubAgent({
        name: r.name,
        model: 'claude-sonnet-4-6',
        systemPrompt: r.systemPrompt,
        tools: ['Read', 'Grep'],
      });

      const result = await agent.run({
        prompt: `${r.prompt}:\n\n${prDiff}`,
        maxTokens: 100000,
      });

      return {
        reviewer: r.name,
        issues: JSON.parse(result.output),
      };
    })
  );

  return results;
}

// 在 CI 中使用
const diff = execSync('git diff main...HEAD').toString();
const reviews = await reviewPR(diff);
const criticalIssues = reviews
  .flatMap(r => r.issues)
  .filter(i => i.severity === 'Critical');

if (criticalIssues.length > 0) {
  console.error('Critical issues found:', criticalIssues);
  process.exit(1);
}
```

## Token Budget 控制

```typescript
const agent = new Agent({
  model: 'claude-sonnet-4-6',
  maxTokens: 500000,       // 单次运行的 token 上限
  timeout: 300000,          // 5 分钟超时
  onTokenUsage: (usage) => {
    if (usage.total > 400000) {
      console.warn('Approaching token limit');
    }
  },
});
```

## SDK vs CLI 选择指南

| 场景 | 推荐 |
|------|------|
| 日常开发中的临时多代理任务 | CLI |
| 需要每天/每 PR 重复运行的审查 | SDK |
| 集成到 CI/CD 流水线 | SDK |
| 探索性的多代理实验 | CLI |
| 产品中嵌入 AI 能力 | SDK |
| 一次性的大型重构 | CLI + Agent Teams |

## 注意事项

1. **SDK 版本与 Claude Code CLI 版本保持同步**
2. **Token 计费与 CLI 使用相同的 API**——不是免费的
3. **安全性**：自定义工具要做好输入验证，特别是数据库查询工具
4. **错误处理**：网络超时、API 限流、token 超限都需要处理
5. **Prompt caching**：SDK 应用应包含 prompt caching 以降低成本
