
# 第二十三章 CLAUDE.md 最佳实践

## CLAUDE.md 是什么

CLAUDE.md 是 Claude Code 的项目级指令文件。每次启动 Claude Code 会话时，它都会自动读取这个文件中的内容作为"背景知识"。

对于 Agent Teams 来说，CLAUDE.md 的影响被放大了——**每个 teammate 都会读取它**。

## 为什么它如此重要

### 单 agent 场景

一个 CLAUDE.md 只被读取一次。即使有 5,000 tokens，也只占用一次上下文。

### Agent Teams 场景

如果有 5 个 teammates，每个都读取 CLAUDE.md：
- 5,000 tokens × 5 = 25,000 tokens 仅用于指令
- 如果 CLAUDE.md 是 10,000 tokens：50,000 tokens

这些 tokens 是"固定开销"——不管任务多简单，都要先付这个代价。

## 2,500 Token 精简法则

社区共识：**CLAUDE.md 控制在 ~2,500 tokens，约一页文本**。

### 什么该放

1. **项目技术栈**（一句话）
2. **关键约定**（不看代码无法推导的规则）
3. **常见陷阱**（新手容易犯的错误）

### 什么不该放

1. **代码规范细节** —— .eslintrc/.prettierrc 已经定义了
2. **项目历史** —— git log 可以查
3. **依赖版本号** —— package.json 已有
4. **目录结构说明** —— Claude 能自己看
5. **通用编程原则** —— Claude 已经知道

## 层级配置

CLAUDE.md 支持三级作用域：

```
~/.claude/CLAUDE.md          # 全局（所有项目）
~/Projects/my-app/CLAUDE.md  # 项目级
~/Projects/my-app/src/CLAUDE.md  # 目录级
```

### 全局级（~/.claude/CLAUDE.md）

放你的个人偏好，适用于所有项目：
```markdown
- 中文沟通，代码可英文
- 回答简洁，结论先行
- 禁止未经确认的 git push
```

### 项目级（项目根目录/CLAUDE.md）

放项目特定的技术约定：
```markdown
## 技术栈
Next.js 14 + TypeScript + Prisma + PostgreSQL

## 约定
- API 路由在 src/app/api/
- 数据库迁移用 prisma migrate
- 测试用 vitest，测试文件与源文件同目录
- 环境变量示例在 .env.example

## 陷阱
- 不要用 `any` 类型
- useEffect 依赖数组不要留空
- Prisma 查询必须在 server component 中
```

### 目录级

极少使用。只在特定子模块有特殊规则时才需要。

## 好的 vs 坏的 CLAUDE.md

### 好的示例（~800 tokens）

```markdown
## Stack
Next.js 14 App Router, TypeScript strict, Prisma, PostgreSQL

## Commands
- `pnpm dev` - dev server
- `pnpm test` - vitest
- `pnpm db:migrate` - prisma migrate dev

## Rules
- API routes: src/app/api/[resource]/route.ts
- Components: PascalCase directories with index.tsx
- Tests: co-located, *.test.ts(x)
- No `any` type, no `@ts-ignore`
- Prisma calls only in server components or API routes

## Gotchas
- Auth middleware in src/middleware.ts uses edge runtime
- Rate limiter state in Redis, not memory
- File uploads go to S3, not local disk
```

### 坏的示例（~5,000 tokens，问题标注）

```markdown
## 项目背景 ← 不需要，Claude 不关心历史
本项目创建于 2024 年 3 月，最初是一个...

## 详细技术栈 ← 太详细，package.json 里有
- React 18.2.0
- TypeScript 5.3.2
- Next.js 14.0.4
...（列了 30 个依赖的版本号）

## 目录结构 ← 不需要，Claude 能自己 ls
src/
├── app/
│   ├── api/
...

## 代码规范 ← 不需要，.eslintrc 里有
- 缩进用 2 空格
- 字符串用单引号
- 末尾加分号
...

## Git 提交规范 ← 放 .commitlintrc
- feat: 新功能
- fix: 修复
...
```

## Agent Teams 专属建议

### 为 teammates 裁剪指令

不是每个 teammate 都需要相同的知识。但由于 CLAUDE.md 是全局加载的，无法针对不同 teammate 定制。

**变通方案**：在 spawn 指令中覆盖特定规则：

```
创建 security-reviewer teammate。
虽然 CLAUDE.md 说测试用 vitest，但你不需要关心这个。
你只需关注安全相关的规则。重点看 OWASP Top 10。
```

### 在 CLAUDE.md 中为团队添加专属指令

```markdown
## Agent Teams 规则
- 修改文件前确认它在你的职责范围内
- 接口变更先发消息给相关 teammate
- 不确定的设计决策发消息给 Lead
```

## 维护 CLAUDE.md

CLAUDE.md 是活文档，需要定期维护：

1. **项目初期**：技术决策频繁，CLAUDE.md 变化大
2. **项目稳定期**：很少更新，偶尔修正
3. **新成员加入**：回顾一下是否还准确

**每季度审查一次**：删除过时的内容，补充新的约定。
