
# 附录 D：Prompt 模板集

本附录整理了全书中所有可直接复用的 prompt 模板。复制粘贴，替换 `[占位符]` 即可使用。


## 一、Cowork 场景模板

### 文件整理

```
帮我智能整理 [目标文件夹] ：
1. 按语义分类（发票/文档/图片/安装包/其他）
2. 对 PDF 读取内容判断类型
3. 创建分类子文件夹并移动文件
4. 生成整理报告到 [输出路径]
5. 不要删除任何文件
```

### 多源笔记合成报告

```
读取 [输入目录] 下所有文件，合成结构化报告：
- 执行摘要（200 字以内）
- 关键发现（按主题分组）
- 行动建议
- 信息来源索引
输出到 [输出路径]，Markdown 格式。
```

### 会议纪要

```
把 [会议记录文件] 转化为结构化纪要：
1. 会议基本信息
2. 讨论要点摘要（每个议题 2-3 句话）
3. 决策事项
4. 行动项表格：| 事项 | 负责人 | 截止日期 | 优先级 |
5. 开放问题
输出到 [输出路径]
```

### 日报/周报

```
基于 [输入目录] 下的笔记，使用 [模板文件] 格式，
生成 [日报/周报]。
1. 提取完成/进行中事项
2. 量化成果
3. 推导下阶段计划
输出到 [输出路径]
```


## 二、Agent Teams 模板

### 创建审查团队

```
创建一个 3 人代码审查团队来审查 [PR 编号/改动描述]：

1. security-reviewer
   - 职责：安全性审查
   - 关注文件：[文件范围]
   - 重点：SQL 注入、XSS、认证绕过、敏感信息泄露

2. perf-reviewer
   - 职责：性能审查
   - 关注文件：[文件范围]
   - 重点：N+1 查询、缺失索引、不必要的计算

3. test-reviewer
   - 职责：测试覆盖率审查
   - 关注文件：[文件范围]
   - 重点：缺失测试、边界情况、mock 合理性

每人完成后：
- 在任务列表标记完成
- 给我发消息，格式：[角色] 发现 X 个问题（N Critical / M High / ...）
- 发现与其他人相关的问题直接发消息给对方
```

### 创建开发团队

```
创建一个 [N] 人开发团队来 [任务描述]：

1. [teammate-name]
   - 负责：[职责]
   - 文件范围：[具体文件/目录]
   - 完成标准：[什么算做完]

2. [teammate-name]
   - 负责：[职责]
   - 文件范围：[具体文件/目录]
   - 依赖：[依赖哪个 teammate 完成]
   - 完成标准：[什么算做完]

[...更多 teammates]

规则：
1. 每人只修改自己范围内的文件
2. 修改接口前先发消息给依赖方确认
3. 完成后在任务列表标记
4. 遇到设计决策先发消息给我
```

### 进入 Delegate Mode

```
进入 Delegate Mode。
从现在开始只做协调，不要自己读文件或写代码。
所有执行工作都交给 teammates。
```

### Plan → Team

```
/plan

[任务描述]

请制定执行计划，包括：
1. 需要新建/修改的文件清单
2. 文件之间的依赖顺序
3. 建议的团队分工和文件边界
4. 风险点
```

审批后：

```
计划批准。进入 Delegate Mode，按计划创建团队并执行。
```

### 带审批的执行

```
在开始实际修改之前，每个 teammate 先提交自己的执行计划：
- 打算修改哪些文件的哪些部分
- 新增/删除/修改的函数和接口
- 预计的测试策略
所有 teammate 提交计划后，我来审批。通过后才能动手。
```


## 三、Subagent 模板

### 并行调查

```
用 [N] 个 subagent 并行调查：
1. subagent-1：[调查任务 A]
2. subagent-2：[调查任务 B]
3. subagent-3：[调查任务 C]
各自独立工作，完成后汇报给我。
```

### 隔离探索

```
用一个 subagent 去 [探索目标]，
只需要告诉我 [需要的结论/数据]。
不需要返回文件内容，只要结论。
```

### 指定模型

```
用 sonnet 模型创建一个 subagent 来 [简单任务]。
```


## 四、安全和约束模板

### Token 预算

```
Token 预算：本次任务总 token 不超过 [数量]。
如果接近预算上限，优先完成最关键的部分。
```

### 退出条件

```
如果 [N] 分钟内没有进展（同一个错误重复出现 3 次），
停止工作并报告当前状态，不要继续尝试。
```

### 文件安全

```
安全规则：
- 不要删除任何文件
- 不要修改 [受保护的文件/目录]
- 只在 [允许的范围] 内操作
- 操作前先告诉我你打算做什么
```
