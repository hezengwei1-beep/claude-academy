
# 第六章 Dispatch：手机远程派任务

> "我在地铁上想到一个任务，掏出手机发给 Claude，到公司时结果已经在桌面等我了。"

## Dispatch 是什么

Dispatch 是 Cowork 的远程派发功能，于 2026 年 3 月推出。简单来说：

**手机端发任务 → 桌面端自动执行 → 手机端查看结果**

你的手机变成了一个遥控器，而你的电脑是执行引擎。所有的 AI 处理、文件操作都在桌面端完成，手机只负责下达指令和查看结果。

## 前置条件

| 条件 | 要求 |
|------|------|
| Claude 计划 | Pro 或 Max（Beta 阶段） |
| 手机 | 安装了 Claude 移动端 App |
| 桌面 | Claude Desktop 保持打开并登录 |
| 电脑状态 | 保持唤醒（不能休眠） |
| 网络 | 手机和桌面都需要联网 |

> **关键限制**：你的电脑必须保持唤醒状态且 Claude Desktop 保持打开。如果电脑休眠了，任务会排队等待，直到电脑恢复。

## 设置步骤

### 1. 更新 Claude Desktop 到最新版

确保桌面端已经更新到支持 Dispatch 的版本。

### 2. 更新 Claude 移动端 App

在 App Store 或 Google Play 更新到最新版。

### 3. 确认账号一致

手机和桌面必须登录同一个 Claude 账号。

### 4. 在桌面端启用 Dispatch

打开 Claude Desktop → Settings → Cowork → 确认 Dispatch 已启用。

### 5. 配置工作文件夹

确保桌面端的 Cowork 已经配置好了工作文件夹和权限。Dispatch 任务使用桌面端已配置的 connectors、plugins 和文件权限。

## 使用流程

### 在手机上

1. 打开 Claude 移动端 App
2. 选择 "Dispatch" 或 "发送到桌面"
3. 描述你的任务
4. 发送

### 在桌面上

1. Claude Desktop 收到任务后自动开始执行
2. 你可以在手机上实时查看进度
3. 执行完成后，结果会同步到手机端

### 对话同步

Dispatch 的对话在手机和桌面间自动同步。你可以：
- 在手机上发起任务
- 在桌面上查看详细执行过程
- 在手机上追问或调整
- 在桌面上继续操作

## 实用场景

### 场景一：通勤路上

在地铁上想到需要整理一批文件：
```
帮我把 ~/Downloads 里今天下载的 PDF 都移到 
~/Documents/项目资料/2026-04/ 并按文件名排序。
```

### 场景二：会议间隙

刚开完会，5 分钟后下一个会，想让 Claude 先处理会议纪要：
```
~/Claude Cowork/INPUTS/ 里有今天的会议录音转写文稿，
帮我按标准模板整理成行动项，输出到 OUTPUTS/。
```

### 场景三：外出时的临时需求

在客户现场，突然需要一份数据：
```
帮我从 ~/Projects/data/ 下的 sales.csv 中，
提取上个月的销售 top 10 产品，做成表格，
发到 ~/Claude Cowork/OUTPUTS/销售报告.md
```

### 场景四：睡前安排明天的任务

```
明天早上我要用到一份竞品分析报告。
请基于 ~/Claude Cowork/INPUTS/竞品资料/ 下的文件，
生成一份对比分析，重点关注定价策略和用户体验。
输出到 OUTPUTS/。
```

## 注意事项与限制

### 电脑必须保持唤醒

这是最常见的"坑"。如果你的 Mac 设置了自动休眠，Dispatch 任务会卡住。

**解决方案**：
- macOS：系统设置 > 电池 > 将"关闭显示器"和"电脑休眠"时间设长
- 或使用 `caffeinate` 命令保持唤醒
- 或在系统偏好中启用"电源小睡"（Power Nap）

### 网络依赖

手机端的指令需要通过网络传递到桌面端。弱网环境下可能有延迟。

### 任务复杂度

Dispatch 适合明确的、可以自主完成的任务。不适合需要频繁交互确认的任务——因为你可能不在电脑旁。

**好的 Dispatch 任务**：
- 文件整理和移动
- 数据提取和报告生成
- 文档格式转换
- 内容生成

**不适合 Dispatch 的任务**：
- 需要你实时审查每一步的敏感操作
- 涉及删除大量文件的高风险操作
- 需要查看 GUI 反馈的 Computer Use 任务

### 安全建议

由于你不在电脑旁，Dispatch 任务的安全性更重要：
- 不要在 Dispatch 中执行删除操作
- 确保工作文件夹权限设置正确
- 敏感文件操作留到你在电脑旁时手动确认

## 进阶：自动化工作流

Dispatch 可以和 Skills 结合，创建固定格式的远程任务：

1. 先在桌面端创建一个 Skill（如"日报整理"）
2. 在手机端通过 Dispatch 调用这个 Skill
3. Skill 按预设的流程自动执行

这样你甚至不需要每次都写详细的指令，一个简短的 Skill 名称就够了。

## Dispatch 工作流设计模板

### 模板一：每日自动化

每天出门前在手机上触发：
```
执行每日整理：
1. 整理 ~/Downloads 中今天的文件
2. 基于 ~/Notes/today/ 的笔记生成日报
3. 输出到 ~/Reports/daily/
```

### 模板二：会议准备

开会前 30 分钟从手机触发：
```
帮我准备 14:00 的项目评审会议：
1. 汇总 ~/Projects/app/ 本周的 git log
2. 统计测试通过率
3. 列出未完成的 TODO
4. 生成一页纸的项目状态摘要到 ~/Reports/
```

### 模板三：应急响应

收到线上告警时从手机触发：
```
紧急：线上报 500 错误。
1. 检查 ~/Projects/app/ 最近的 git log
2. 看有没有最近的配置变更
3. 分析可能的原因
4. 给出初步排查建议
不要修改任何代码，只分析。
```

### 模板四：信息收集

出门在外需要数据时：
```
帮我从 ~/Data/sales-2026.csv 中提取：
1. 本月销售 Top 5 产品
2. 环比增长率
3. 生成简要摘要（200 字以内）
输出到 ~/Reports/quick-stats.md
```

## Dispatch 的限制和变通

| 限制 | 变通方案 |
|------|---------|
| 桌面必须醒着 | macOS：系统设置 → 电池 → 显示器关闭时间设长 |
| 不能做需要确认的操作 | 任务描述中写清所有决策，不留歧义 |
| 长任务可能中断 | 拆成多个短任务 |
| 不能看到 GUI 操作 | Dispatch 不支持 Computer Use，只能操作文件 |

## Dispatch + Skills 的组合

最强大的用法：事先创建好 Skills，从手机一句话触发：

```
# 手机端输入
运行"周报生成"

# 桌面端自动执行预定义的 Skill 流程
# 不需要每次都写详细指令
```

这把 Dispatch 的效率最大化——手机端只需 5 个字，桌面端执行完整的多步骤工作流。
