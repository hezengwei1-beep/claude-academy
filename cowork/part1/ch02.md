
# 第二章 Claude 协作体系全景

## 四层架构：一图看懂

Claude 的协作能力不是单一功能，而是一个分层体系。从"最简单易用"到"最强大灵活"，可以分为四层：

```
┌─────────────────────────────────────────────────┐
│  Layer 4: Git Worktrees                         │
│  完全隔离的并行会话，独立分支独立代码              │
│  面向：需要独立分支开发的开发者                    │
├─────────────────────────────────────────────────┤
│  Layer 3: Subagents（子代理）                    │
│  单会话内的并行工作者，向主代理汇报                │
│  面向：开发者（默认可用）                         │
├─────────────────────────────────────────────────┤
│  Layer 2: Agent Teams（代理团队）                 │
│  多 Claude 实例组成团队，互相通信协作              │
│  面向：开发者（实验性功能，需手动启用）             │
├─────────────────────────────────────────────────┤
│  Layer 1: Claude Cowork（桌面自动化）             │
│  内置于 Claude Desktop，操作本地文件和应用         │
│  面向：所有用户（非技术用户友好）                  │
└─────────────────────────────────────────────────┘
```

让我们逐层解析。

## Layer 1: Claude Cowork —— 桌面自动化代理

**一句话定义**：内置于 Claude Desktop 应用的 AI 代理，能直接操作你电脑上的文件和应用。

**面向人群**：所有付费用户（Pro/Max/Team/Enterprise），无需技术背景。

**核心能力**：
- 自动整理文件和文件夹
- 合成和格式化文档
- 提取和清洗数据
- 研究综合与报告生成
- 通过 Computer Use 直接操作桌面应用的 GUI

**使用方式**：在 Claude Desktop 应用中，从 "Chat" 切换到 "Cowork" 标签，选择工作文件夹，描述任务即可。

**独特功能**：
- **Dispatch**：从手机远程派发任务到桌面
- **Computer Use**：让 Claude 直接控制你的鼠标和键盘操作应用
- **Skills**：可复用的提示词模板

**局限**：不适合复杂的软件开发任务，这是 Claude Code 的领域。

## Layer 2: Agent Teams —— 多代理团队

**一句话定义**：多个 Claude Code 实例组成团队，有 Leader 协调，队友之间可以直接通信。

**面向人群**：有开发经验的用户。

**核心组件**：
| 组件 | 作用 |
|------|------|
| **Team Lead** | 主会话，创建团队、分配任务、综合结果 |
| **Teammates** | 独立 Claude Code 实例，各有独立上下文 |
| **Task List** | 共享任务列表，支持依赖和自动认领 |
| **Mailbox** | 代理间直接通信的收件箱系统 |

**关键特性**：队友之间可以互相发消息、共享发现、质疑彼此的结论。这是 Agent Teams 区别于 Subagents 的核心。

**启用状态**：实验性功能，需设置环境变量 `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`。

**典型用例**：
- 并行代码审查（安全 + 性能 + 测试覆盖率）
- 竞争假设调试（多个 agent 各自调查不同假设）
- 跨层协调开发（前端 + 后端 + 数据库）

## Layer 3: Subagents —— 子代理

**一句话定义**：单个 Claude Code 会话内部生成的并行工作者，完成后向主代理汇报结果。

**面向人群**：开发者。

**与 Agent Teams 的关键区别**：

| 维度 | Subagents | Agent Teams |
|------|-----------|-------------|
| 通信 | 只向主代理汇报 | 队友间可直接通信 |
| 协调 | 主代理管理 | 共享任务列表 |
| 关系 | 上下级（层级） | 同事（扁平） |
| 启用 | 默认可用 | 需手动启用 |

**使用方式**：
- `/agents` 管理界面
- 自然语言："用一个 subagent 调查认证模块的性能问题"
- `@-mention`：`@code-reviewer 查看这个 PR`

**典型用例**：
- 隔离的研究任务（避免污染主 agent 上下文）
- 快速并行数据收集
- 代码审查辅助

## Layer 4: Git Worktrees —— 独立工作树

**一句话定义**：用 Git 的 worktree 功能创建完全独立的代码副本，每个副本运行一个独立的 Claude Code 会话。

**面向人群**：需要同时开发多个功能或修复多个 bug 的开发者。

**核心特点**：
- 每个 worktree 有独立的分支和文件系统
- 会话之间完全隔离，互不影响
- 无需额外配置，开箱即用

**使用方式**：
```bash
# 创建并进入工作树
claude --worktree feature-auth
claude --worktree bugfix-123
```

**典型用例**：
- 同时开发新功能和修复紧急 bug
- 独立探索不同的技术方案
- 长期运行的重构与日常开发并行

## 与竞品的对比

Claude 的协作体系在市场上并非孤立存在。以下是与主要竞品的横向对比：

| 维度 | Claude Cowork | Claude Code Agent Teams | ChatGPT Agent | GitHub Copilot | Google Jules |
|------|--------------|------------------------|---------------|----------------|-------------|
| 运行位置 | 本地桌面 | 本地终端 | 云端 | 云端 | 云端 |
| 多代理 | 否（单代理） | 是（多代理团队） | 否 | 否 | 否 |
| 面向人群 | 非技术用户 | 开发者 | 所有用户 | 开发者 | 开发者 |
| 文件操作 | 本地文件系统 | 本地代码仓库 | 虚拟浏览器 | GitHub 仓库 | GitHub 仓库 |
| 独特优势 | Dispatch + Computer Use | 代理间通信 + 共享任务 | 网页自动化 | IDE 集成 | 异步 PR |

**关键差异**：Claude 的核心优势在于**本地执行**和**多代理协作**。所有操作都在你的机器上执行，数据不离开本地。而 Agent Teams 的代理间通信能力，目前在竞品中还没有对应产品。

## 全景视角

把这四层放在一起看，Claude 提供了一个从"零技术门槛"到"专家级多代理编排"的完整谱系：

```
简单 ◄──────────────────────────────────────────► 强大
Cowork ──► Subagents ──► Agent Teams ──► Worktrees
非技术用户    开发者入门     进阶开发者      专家级并行
```

你不需要掌握全部四层。根据你的角色和需求，找到最适合的那一层，深入使用即可。下一章的决策树会帮你做出选择。
