
# 第三章 我该用哪个？决策树

选择困难症？这一章给你一个 60 秒做出决策的框架。

## 决策流程图

```
你的任务是什么？
│
├─► 非代码任务（文件整理、文档、研究、数据处理）
│   └─► ✅ Claude Cowork
│       （Part 2 详解）
│
├─► 代码任务，但只需要一个 agent
│   └─► ✅ 单个 Claude Code 会话
│       （本书不重点覆盖，参考 Claude Code 官方文档）
│
└─► 代码任务，需要多个 agent 并行
    │
    ├─► 任务之间完全独立，不需要任何通信？
    │   │
    │   ├─► 需要独立的 Git 分支？
    │   │   └─► ✅ Git Worktrees（Ch16）
    │   │
    │   └─► 不需要独立分支，只是想并行？
    │       └─► ✅ /batch 或 Subagents（Ch15）
    │
    ├─► 需要隔离输出，结果汇报给主 agent？
    │   └─► ✅ Subagents（Ch15）
    │
    └─► 需要 agent 之间讨论、协调、共享发现？
        └─► ✅ Agent Teams（Ch9-Ch14）
```

| 场景 | 推荐工具 | 原因 |
|------|----------|------|
| 整理 Downloads 文件夹 | Cowork | 非代码，文件操作 |
| 合成多个笔记为报告 | Cowork | 非代码，文档处理 |

| 同时开发新功能和修 hotfix | Worktrees | 完全独立的分支 |
| 重构大型模块（多文件协调） | Agent Teams | 需要 agent 间协调文件边界 |
| 批量修改 20 个独立文件 | /batch | 完全独立的并行操作 |

## 成本对比

成本是选择工具时不可忽略的因素：

| 工具 | 相对成本（以单 agent 为 1x） | 说明 |
|------|------------------------------|------|
| 单个 Claude Code | 1x | 基准 |
| Subagents | 1.5-2x | 每个 subagent 有独立上下文，但通常任务较短 |
| Agent Teams（3 人） | 3x | 每个 teammate 维护完整独立上下文 |
| Agent Teams（5 人 + Plan Mode） | 5-7x | Plan Mode 下每个 teammate 的上下文消耗更大 |
| Worktrees | 1x × N | 每个 worktree 是独立会话，成本线性叠加 |
| Cowork | 按 Claude 付费计划 | Pro $20/月，Max 更高，不按 token 计费 |

**经验法则**：如果任务用单个 agent 能在合理时间内完成，就不要上多 agent。多 agent 的价值在于处理单 agent **无法高效完成**的任务。

## 常见误区

### 误区一："所有任务都该用 Agent Teams"

**现实**：Agent Teams 适合需要代理间讨论和协调的复杂任务。简单的并行操作用 Subagents 或 /batch 更高效、更便宜。

### 误区二："Agent 越多越快"

**现实**：协调成本随 agent 数量非线性增长。社区实践的甜区是 **3-5 个 teammates**。超过这个数量，边际收益递减，冲突风险递增。16 个 agent 的 C 编译器项目是特殊场景（大量天然独立的测试修复），不是常态。

### 误区三："Cowork 是 Claude Code 的简化版"

**现实**：它们是完全不同的产品。Cowork 是桌面自动化代理，面向非技术用户，操作文件和应用。Claude Code 是编码代理，面向开发者，操作代码仓库。两者互补，不是替代关系。

### 误区四："用了多 agent 就不需要人看了"

**现实**：多代理是 **supervised workflow**（有监督的工作流）。你是项目经理，agent 是团队成员。你需要定期检查进度、审查质量、做方向决策。放任不管的案例在 Part 5 中有详细的失败教训。

## 混合模式

实际工作中，你经常需要组合使用多种工具：

### 组合一：Agent Teams + Subagents

Team Lead 使用 Agent Teams 协调三个 teammates，每个 teammate 内部又用 Subagent 做隔离研究。这样既有团队层面的协调，又有个体层面的高效探索。

### 组合二：Worktrees + 单 agent

在两个 worktree 中各运行一个 Claude Code 会话。一个做新功能开发，另一个做 bug 修复。两者完全独立，互不干扰。

### 组合三：Cowork + Claude Code

用 Cowork 处理非代码任务（文档整理、研究综合），用 Claude Code 处理代码任务。两者各司其职。

## 快速决策清单

在你启动多 agent 之前，问自己这五个问题：

1. **单 agent 能做吗？** → 如果能，就不要上多 agent
2. **任务之间有依赖吗？** → 有依赖用 Agent Teams，无依赖用 Subagents/batch
3. **agent 需要互相讨论吗？** → 需要用 Agent Teams，不需要用 Subagents
4. **预算允许吗？** → 3 个 teammates ≈ 3x 成本
5. **你有时间监督吗？** → Agent Teams 需要你做项目经理

如果五个问题都考虑清楚了，你就可以自信地选择合适的工具。接下来的章节会手把手教你使用每一种工具。
